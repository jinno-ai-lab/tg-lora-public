# 10. Progressive Freezing + Activation Matching — 設計書

> **位置づけ**: GOAL §1.6 / §3.1 の技術的詳細。なぜこの手法が機能するのか、既存の観測とどう整合するのか、何が制約になるのかを正確に説明する。

## 1. 何をするのか（What）

少データ・多エポックの LoRA 学習の途中で、後段（出力側）のレイヤから順にフリーズ（重み固定）していく。フリーズした層は学習対象から外れ、backward 計算が削減される。フリーズのタイミング・順序・深度をスケジュールとして設計する。

## 2. なぜ計算が減るのか（Why computation decreases）

通常の full backprop では、loss から入力側まで全レイヤを逆向きに貫通する。backward の計算量は勾配が通過するレイヤ数に比例する。

フリーズには2つのレベルがある:

**Level 1 — Weight gradient 停止（基本）**

フリーズした層の重み勾配をゼロにする。activation 勾配は後段から前段へ伝播し続けるが、フリーズ層では「重み勾配の計算」と「optimizer によるパラメータ更新」が不要になる。また optimizer が保持するモメンタム・分散の状態を解放できるため VRAM も減る。activation 勾配の伝播自体は残るため、backward グラフの切断というより「勾配計算のうち重い部分（optimizer step）の省略」に近い。

**Level 2 — Activation gradient も切断（発展）**

フリーズした層を backward グラフから完全に外す。前段の層は、最終 loss から逆伝播してくる勾配ではなく、**フロント層だけの local loss** で学習する。この local loss の target が `xin`（後述）。

Level 1 は既存の PyTorch の機能（`requires_grad=False`）で即座に実装できる。Level 2 は backward グラフの構造変更が必要だが、計算削減は最大になる。

## 3. なぜ学習信号が生まれるのか（Why learning signal exists）

これが最も重要な論理ポイント。「後段を固定して前段だけを学習」したとき、前段は何に向かって改善されるのか。

### 3.1 xin の定義

レイヤ X をフリーズする時点での、**レイヤ X の入力活性**を `xin` と呼ぶ。各データポイント x ごとに観測されるベクトル:

```
xin(x) = Layer X の直前の活性 = Layer X-1 の出力（フリーズ時点）
```

この `xin` をすべての学習データについて記録（キャッシュ）する。

### 3.2 xin が運ぶ情報

`xin` は単なる「たまたまその時の出力値」ではない。次の条件をすべて満たす:

1. **後段が正しく機能するための入力である**: レイヤ X 以降は、フリーズ時点での学習結果として、`xin` を受け取ったときに良い出力を出すことが保証されている。フリーズした後段はもう変わらない。つまり後段が前提としている入力がまさに `xin`。

2. **タスク情報を含んでいる**: `xin` は、full 学習の loss に基づいて後段が適応した結果として生じた値。後段が「loss を下げるために最適化した入力」に対して出した応答の入力側。最終 loss の情報が `xin` の中に間接的に埋め込まれている。

3. **現在の Layer X-1 の出力とは異なる**: これが鍵。Layer X-1 はまだ最適化の途中であり、`xin` を正確に出力できていない。`xin` は「成熟した後段が要求する入力」であり、現在の X-1 は「未熟な状態で出している別の値」。この差が learning signal になる。

### 3.3 「現状維持」にならない理由

「target = 現在の X-1 の出力」であれば、loss は常にゼロになり、学習信号は生まれない。しかし target は `xin` = 「フリーズ時点での X の入力」であり、これはフリーズという**過去の時点**での値。

- 時刻 t でフリーズ → `xin` を記録
- 時刻 t+1 以降、X-1 を `xin` に近づけるよう学習
- 時刻 t での X-1 の出力 ≠ 時刻 t+1 での X-1 の出力（学習が進むから）
- `xin`（target）は固定されているから、X-1 の出力が変化するにつれて loss が動く

`xin` と「現在の X-1 の出力」の差が、学習を駆動する勾配になる。

### 3.4 なぜ逆行列が不要なのか

「X-1 を学習して、X が良い出力を出すような入力を出させる」と聞くと、X の逆写像 f⁻¹ を解く必要があるように思える。しかし実際には:

- `xin` は **forward で実際に観測された値** であり、計算で求めたものではない
- キャッシュから取り出して target として使うだけ
- 逆算も近似逆写像も不要

この点は、可逆性を仮定する Bornschein et al. (2021) やlayer-wise training の一部系統とは異なる。`xin` は「正解」を逆算したものではなく、「正解が与えられたときに観測された実測値」である。

## 4. 単一 run 内でどう進行するか（How it proceeds within a single run）

### 4.1 全体像

```
Epoch 0 ────────── T1 ────────── T2 ────────── T3 ──────→
[    全層 full 学習    ][段階1 ][  段階2  ][段階3  ]
                         ↓       ↓          ↓
                       X を     X-1 を     X-2 を
                       フリーズ  フリーズ    フリーズ
```

- **T1 まで**: 全層を full backprop で学習。この期間で後段が十分に成熟する。
- **T1 で**: レイヤ X をフリーズ。直前の forward で `xin(X)` をキャッシュ。
- **T1〜T2**: レイヤ X-1 を `xin(X)` を target に local loss で学習。X の backward は省略。
- **T2 で**: X-1 も成熟したと判断 → X-1 をフリーズ。`xin(X-1)` をキャッシュ。
- **T2〜T3**: レイヤ X-2 を `xin(X-1)` を target に local loss で学習。X, X-1 の backward は省略。
- 以降、同様に前段へ広げていく。

### 4.2 なぜ「二重学習」ではないのか

target を生成するために別の full run が必要だと思うかもしれない。しかし:

- `xin` は「同じ run の中で、フリーズした瞬間に forward で観測された値」
- 別の run から取ってくるのではない
- 学習の進行そのものが target を生成する
- 別 run のコストを償却する問題は発生しない

### 4.3 なぜ少データ多エポックで効くのか

同じデータを何度も見る（エポックを重ねる）前提だから:

- 後段は早い段階で「このデータに対する自分の役割」を固め終える
- 固め終えた後段を、残りのエポックにわたって backward し続けるのは無駄
- フリーズして前段に計算を集中させれば、後半エポックが軽くなる
- エポック数が多いほど、フリーズ以降の恩恵が大きい

1回しかデータを通さない大規模事前学習では、フリーズする余裕（後段が固まるまでの時間）がないため、この手法の対象外になる。

## 5. 既存の観測との整合（Why existing observations support this）

### 5.1 層間非整列 (cos ≈ 0) → フリーズの安定性の根拠

これまでの TG-LoRA の解析で、異なる層の更新方向の間のコサイン類似度が ≈ 0（無相関）であることが確定している。

これは次を意味する:

- **各層は独立に学習している**: 層 A の更新方向は層 B の更新方向と無関係
- **フリーズが前段に与える影響が小さい**: 層 X をフリーズしても、その前段 X-1 の最適な更新方向は X の状態に依存しない（依存するのは target xin のみ）
- **xin が安定**: 層 X がフリーズされた後、X-1 が学習して出力が変化しても、X の入力分布が大きく崩れることはない。層間の依存が弱いため、局所的な変化が連鎖しない

層間依存が強い場合、X-1 の出力が変化すると X の振る舞いも変わってしまい、固定した `xin` が target として不適切になる。しかし層間が独立なら、X-1 の変化は X を通じて後段に影響しない。

### 5.2 cycle 6 相転移 → フリーズタイミングの設計指針

全層が一斉に挙動を変えるフェーズ遷移が cycle 6 で観測されている。

これはフリーズのタイミング設計に直接効く:

- **相転移前**: 後段はまだ落ち着いていない。この時点でフリーズすると、`xin` は「未熟な後段が要求する入力」になり、品質が低い
- **相転移後**: 後段の役割が一気に確定する。ここでフリーズすれば、`xin` は「成熟した後段が要求する入力」になり、品質が高い
- **大幅後**: 過学習のリスク。`xin` は特定の局所解に過剰適応している可能性

相転移の前後でフリーズタイミングを変えたときの valid_loss を比較することが、Phase 2 のタイミングスイープの核心になる。

### 5.3 初期安定（attention の rank-1 方向）→ フリーズ順序のヒント

attention 層は早い段階（cycle 0〜5）で rank-1 方向が安定する。これは「attention 層は先にフリーズしてもよい」ことを示唆する。一方 FFN 層は長く安定しない可能性がある。

これが「収束順フリーズ」（安定した層から固める）の候補と「後方素直フリーズ」（出力側から機械的に固める）のトレードオフになる。

### 5.4 rank-1 支配 → xin の再現の見通し

各テンソルの更新が1つの方向に集中している。フリーズ後もその方向は安定しやすい。X-1 が `xin` を再現する難易度が、ランダムな分布を再現するより低い可能性を示唆する。

## 6. 制約と限界（Constraints and limitations）

### 6.1 フリーズのタイミング

フリーズする時点で、後段がすでに十分成熟している必要がある。早すぎると:

- 未熟な後段の `xin` を target にしてしまう
- 未熟さが「正しい入力」としてキャッシュされ、前段がそれに追従する
- 品質劣化が前段へ伝播

「十分成熟した」の判定が設計の最大の自由度であり、相転移観測がその判断材料になる。

### 6.2 代理ロスの限界（Level 2 固有）

`xin` を target にした MSE は、最終タスク loss の代理 (proxy) である。MSE がゼロ（X-1 が `xin` を完全に再現）でも、最終タスク品質が同じとは限らない:

- MSE は点ごとの一致を測るが、分布の高次構造（共分散、相関）は捉えない
- 後段が入力の分布の微妙な変化に敏感な場合、`xin` の点一致では不十分
- 層間非整列が強ければこの問題は小さいが、ゼロではない

MSE 以外の損失関数（cosine loss、分布一致 KL など）で改善する可能性があるが、これは Phase 3 のアブレーションで検証する。

### 6.3 1回学習には効かない

少データ多エポックが前提。1エポックで終わる学習ではフリーズする余裕がなく、この手法の対象外。

## 7. 具体的な計算削減の見積もり [UNVERIFIED]

### Level 1（weight gradient 停止）

| 項目 | 削減内容 |
|------|----------|
| 重み勾配計算 | フリーズ層分が不要 |
| optimizer 更新 | フリーズ層分が不要 |
| optimizer 状態 (VRAM) | フリーズ層のモメンタム・分散状態を解放 |
| activation 勾配 | **削減なし**（最終 loss から前段へ伝播） |

削減量はフリーズ層の割合に比例。後段 25% をフリーズすれば、その 25% 分の weight gradient + optimizer step が消える。

### Level 2（activation gradient も切断）

| 項目 | 削減内容 |
|------|----------|
| 重み勾配計算 | フリーズ層分が不要 |
| optimizer 更新 | フリーズ層分が不要 |
| optimizer 状態 (VRAM) | フリーズ層の状態を解放 |
| activation 勾配 | **フリーズ層を通る伝播が不要** |

Level 1 に加えて、backward グラフの suffix が物理的に切れる。フリーズ層を通る activation 勾配の計算が丸ごと消えるため、削減は Level 1 より大きい。ただし代理ロスの整合性リスクを伴う。

## 8. 実装に必要な構成要素 [UNVERIFIED]

1. **フリーズスケジュール**: いつ・どの層をフリーズするかを制御する設定。エポック数・層インデックス・収束判定条件のいずれかで駆動。
2. **xin キャッシュ**: フリーズ時点での対象層の入力活性を、全学習データについて記録。`src/tg_lora/activation_cache.py` が基盤として使える。
3. **Local loss 関数**: MSE・cosine・分布一致のいずれか。Level 2 で必要。
4. **層別勾配制御**: Level 1 は `requires_grad=False` で実現。Level 2 は backward グラフの構造変更（detach 等）が必要。
