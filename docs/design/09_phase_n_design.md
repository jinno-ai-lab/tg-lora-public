# TG-LoRA N最適化設計仕様（Phase-N）

**前提条件（着手の発火条件）**

本設計は、EMA方向差し替え（v0をraw history平均からVelocity EMA方向へ置換）が完了し、α=1の純粋外挿でtraining batch損失の爆発（最大12.6）が解消され、EMA方向に1単位進んでlossが下がることが実測で確認された後に着手する。この前提が満たされない限り、本設計には進まない。方向が信頼できない状態でNを最適化しても土台がないためである。

## 1. 問題定義：取引コスト込みの純リターン最大化

N最適化は「予測問題」ではなく「コスト込みの最適化問題」として扱う。株価運用における取引コスト控除後の純リターン最大化と同型である。重み軌跡が決定論的（答えが動かない）である点で株価より易しく、確率的予測器は不要である。

通貨はすべて backward 回数換算で統一する。

- **粗利益（Return）**: 外挿が accept された際に節約した backward 回数。N単位 accept で N×K 回分。
- **確認コスト（Cost）**: 止め時判定・答え合わせに使う forward。backward換算で forward≈0.5 backward。
- **損切りコスト（Risk）**: reject 時に丸損する確認 forward。pilot の backward は別途計上済み（削減ゼロ）。

最大化目標：

純リターン=accept∑​(N⋅K)−∑(確認forward×0.5)−reject∑​(無駄forward×0.5)

**設計上の含意（重要）**: backward節約（N×K、Kあたり数回×N単位）は確認forward（数回×0.5）より桁で重い。したがって確認コストをケチって reject を増やすより、適度に確認コストを払って accept 率を上げる方が純リターンは大きい。最適点は「確認を最小化する」側ではなく「reject を最小化しつつ確認を必要十分に抑える」側にある。

## 2. 中核設計：適応的トレンドフォロー外挿

Nを事前推定しない。EMA方向に追従しながら、トレンドの生死に応じて進み、止めた地点の単位数を実効Nとする。これは当初設計の「区分外挿」、株価の「トレンドフォロー」、本調査の実測（事前指標cosは識別力なし）の三者が収束する形である。

### 2.1 単位前進

外挿方向は EMA単位方向 v^EMA​（差し替え後）。1単位の前進量は、理論で確定したスケール ∥Δ1step​∥（直前1サイクルの1ステップあたり実移動量のノルム）とする。

W←W+s⋅∥Δ1step​∥⋅v^EMA​

歩幅係数 s はボラティリティ（軌跡の局所曲率）で調整する。トレンドが直線的（低ボラ）なら s を大きく、不安定（高ボラ）なら小さく。初期は s=1 とし、後述の監視指標で動的調整する。

### 2.2 二段構えの止め時判定（コスト最適化の核心）

止め時判定を、無料の代理指標による常時監視と、有料の精密確認の二段に分ける。これが本設計の最重要のコスト最適化である。

**一次フィルタ（無料・常時）**: EMA方向の向き変化を監視する。EMAは逐次更新されるため、1単位進む前後でのEMAベクトルの向きの変化（前ステップEMAと現ステップEMAのcos）は、追加forwardなしで観測できる。この向き変化がトレンドの曲がり（曲率）を表す代理指標となる。

- EMA向き変化のcosが安定（高い）→ トレンドは生きている。確認を省いて次の単位へ進む（コスト節約、利を伸ばす）。
- EMA向き変化のcosが閾値を割る（曲がり始め）→ 危険信号。二次確認を発動。

**二次確認（有料・危険時のみ）**: 一次フィルタが危険信号を出したときだけ、クイック評価セット（32件）で実際の loss を測定し、accept/reject 相当の精密判定を行う。loss が改善していれば続行、悪化していれば止めてロールバック。

この二段構えにより、確認 forward を「トレンドが曲がりかけた危険局面」に集中投下し、安定局面では完全に無料で進める。常時有料確認（毎単位32 forward）に比べ、確認コストを大幅に削減する。

### 2.3 非対称リスク管理（利は伸ばし、損は切る）

- **利を伸ばす**: 一次フィルタが安定を示す間は、確認を省いて単位を積み増す。上限Nは設けるが（暴走防止のハードキャップ、初期値N=20）、トレンドが生きる限りそこまで伸ばしてよい。
- **損を切る**: 二次確認で悪化が判明したら即座に止め、直前の最良単位地点までロールバックする。深追いしない。

### 2.4 スケールインによる損切りコスト抑制

二次確認の発動時、いきなり全Nをコミットして答え合わせするのではなく、現単位までの外挿を段階的にコミットする。小さいNで通れば追加で伸ばし、外れても捨てる量が小さい。reject時の丸損（§1のRisk）を構造的に小さく保つ。

一次フィルタ閾値は、前回判明した「predicted_consistencyのレンジが0.77〜0.84と狭い」問題を踏まえ、固定マジックナンバーを置かず、着手後の実測EMA向き変化分布から識別力のある切り方を決める。株価テクニカルのマジックナンバー（14日等）をそのまま持ち込まず、構造のみ借用しパラメータは重み軌跡で測り直す原則に従う。

## 4. 効率会計への反映

本設計は確認forwardを増やす方向であり、§3の効率会計（reduction_rate）に影響する。一発外挿（確認最小・当たらず実効削減ゼロ）から、適応追従（確認やや増・accept率向上で実効削減プラス）へのトレードオフである。当初設計が一発外挿に逃げた理由が確認コストであったが、一発外挿が機能しない（ランダム方向に飛ぶ）と判明した以上、確認コストを払って実際に効く追従を取る判断は既定路線とする。reduction_rate は確認forwardを backward換算で差し引いた純値で報告する。

## 5. 検証指標（着手後に測る）

着手後の検証runで報告する数値：accept率（本番相）、純リターン（§1の式）、一次フィルタが二次確認を発動した回数と発動時のreject率（一次フィルタの識別力）、実効Nの分布（事前推定なしで結果的にどこまで伸びたか）、確認forward総数のbackward換算、同一budget点でのBaseline比較。判定基準は前回同様、reduction_rate（純値）とloss改善が両立する場合のみ「成功」とする。

## 6. スコープ外（本フェーズで触らない）

EMA方向差し替えそのもの（前提条件であり別フェーズ）、FDフィットの係数最適化（α=1固定で進める。係数最適化はN最適化が機能した後の精緻化フェーズに回す）、raw history記録の粒度・rollback記録問題（計測正確性の既知課題として記録のみ、優先度低）。

---

**設計判断の要約（なぜこの一案に決めたか）**

迷いどころだった「止め時判定」は、二段構え（無料の一次フィルタ＋危険時のみ有料の二次確認）に確定しました。理由は、あなたの主眼であるコスト最小化に最も忠実だからです。毎回有料確認（実装は単純だがコスト高）でも、確認なし一発（コスト最小だが当たらず実効ゼロ）でもなく、無料の代理指標で常時見張り危険時だけ課金する、が純リターン最大化の最適点です。「Nを事前推定せず追従の結果として決める」も確定としました。実測でcosが識別力を持たないと判明した以上、事前推定の土台がなく、株価の枯れた知恵（予測せず追従する）に倣うのが最善です。
