# 07_design_decisions_log.md — 設計判断の履歴

## 1. 事実 (Facts)

### 1.1 アルゴリズムの発展史とアーキテクチャの変遷

1.  **TG-LoRA コアの初期実装 (2026-05-20, `commit f018801`)**:
    *   **事実**: 勾配速度追跡（EMA velocity）による外挿、悪化時のロールバック（`rollback_manager`）、および hill-climbing ランダムウォーク（`random_walk_controller`）によるハイパーパラメータ探索を組み合わせた最初期のループが実装された。
2.  **速度アノマリーとトレンド追跡の導入 (2026-05-21, `commit bbcb7e7`)**:
    *   **事実**: `Velocity` 状態に EMA 更新後の L2 ノルム履歴（`magnitudes`）を記録するプロパティを追加。過去の履歴に基づく統計的スパイク検知関数 `is_magnitude_anomalous()`（平均 ＋ 3.0 $\times$ 標準偏差）および線形回帰による速度の加減速傾向の判定関数 `magnitude_trend()` を実装した。
3.  **Linearity Guard (Baseline-like Fallback) の導入 (2026-05-28, `commit 252cbda`)**:
    *   **事実**: 速度アノマリーの検知（Trigger A）、正の加速度（Trigger B）、または過去の低承認率（Trigger C）が発生し、かつ直近のパイロットロスが不安定な場合に、外挿試行を完全にスキップして $N=0$ として通常の QLoRA 更新のみをモデルに適用するフォールバック安全装置 `_should_fallback_to_baseline_like` が実装された。
4.  **コサイン類似度に基づく $N$ 決定と検証スキップ (2026-06-02, `commit 5857db0` および `commit e37fa00`)**:
    *   **事実**: 短期方向（$dW$）と長期的な速度ベクトル（$v_0$）の類似度（EMA consistency）に基づき、外挿ステップ数 $N$ を動的に決定する `choose_N` のロジックを追加。類似度が高い場合に、答え合わせのバリデーションコストを省略するゲート `cosine-gated validation skip` が実装された。
    *   **現在のステータス**: M9が有効な場合、このレガシーな検証スキップロジックはバイパスされる。かつてはM9パス独自のフィットバッチ（`m9_batch`）による即時判定および検証スキップ（`subspace_m9_fitted`）に置換されていたが、その後 §1.1-6 のデータリーク解消に伴い、独立検証セット（`valid_quick_loader`）による判定へ移行したため、現在は検証スキップは行われず必ず独立検証が実施される。
5.  **Prior-based Subspace Learning (M9) への移行 (2026-06-05, `commit a5531d3` および `commit 41d1894`)**:
    *   **事実**: 1次元の直線探索（手探りの $\alpha$ スケール調整）およびバリデーションのオーバーヘッドによる効率の頭打ちを是正するため、勾配軌跡から Gram-Schmidt 直交化により直交基底 $\{v_0, u_1, u_2\}$ を構築し、部分空間上で有限差分（FD）を用いて 1ステップで最適な座標係数（$\alpha, \beta_1, \beta_2$）を特定する M9 を実装した。
6.  **M9 外挿 $N$ 倍スケーリングとデータリークの解消 (2026-06-06, 現在のブランチ)**:
    *   **事実**: $N$ ステップ外挿時にパラメータ空間的支配方向 $v_0$ が $N$ 倍されていなかったバグを修正し、物理移動量と削減率の整合性を確保した。また、有限差分フィット時の訓練ミニバッチ（`m9_batch`）を accept/reject 判定の `loss_after` 評価に再利用していたデータリークを解消し、独立した検証セット（`valid_quick_loader`）による客観的なオンライン判定へ完全移行した。

### 1.2 却下・廃止された設計案 (Rejected Alternatives)

*   **直線探索 (Line Search) / 手探り $\alpha$ スケール調整の廃止**:
    *   各サイクルで validation loss を繰り返し計算して最適な外挿倍率を探索していた初期のアプローチは、バリデーションにかかる Forward パスのオーバーヘッドが大きく、かつ進行方向の急変に追従できず効率が 1.24倍付近で頭打ちとなったため、M9 部分空間への移行時に廃止された。線形探索（逐次探索）を非優先（廃止）とした理由は、逐次的な探索がサイクルあたり forward を増やし高速化目的に反するためである。
*   **生 prior の直接適用案の却下**:
    *   有限差分による係数フィットを行わず、単に過去の $v_0$ をそのまま $\alpha=1.0$ 固定で外挿適用する案が検討されたが却下された。その理由は、prior 方向をそのまま適用すると個別サイクルの曲率・残差成分を無視するため、FD フィットで係数（alpha, beta1, beta2）を最適化する経路を採用した。
*   **wall-clock 速度（時間）の前面表示・主張の廃止**:
    *   ハードウェア環境や validation 評価時間の割合に大きく依存するため、提案手法の評価指標から「実行時間 (wall-clock)」を排除し、環境非依存で客観的な「実 backward 削減率」のみを前面に押し出す方針に決定した。

## 2. 設計意図 (Rationale)

*   **なぜ直線探索から M9 Subspace Learning へ移行したのか**:
    *   手探りの直線探索は、学習の進行に伴って方向ベクトルが変化する局面に追従できず、スケール調整の誤差が蓄積して効率が頭打ちになっていた。部分空間を構成して直交化 PC を加えることで、進行方向の微修正（ステアリング）を許容しつつ、有限差分で一発で最適な係数をフィットできるため。
*   **なぜ accept 判定用の validation サイズを 32 件としたのか**:
    *   計算コストを抑制し高速化の目的を維持するため。また、M9の3次元部分空間（v0, u1, u2）への探索空間制限自体が強い正則化として作用するため、32サンプルという極小データでも過適合を強力に抑制し判定ロバスト性を十分に担保できるため（[04_acceptance_and_validation.md](file:///home/jinno/tg-lora/docs/design/04_acceptance_and_validation.md) の Rationale に詳述）。

## 3. 追加判断ログ (2026-06-06) — M9 外挿の根幹思想と計算量トレードオフ

### 3.1 意思決定の要約

*   **採用した設計**:
    *   低次元部分空間への圧縮 ＋ 方向上での係数フィット（backward 非消費の外挿）。
*   **却下した代替案1**:
    *   $N$ サイクルの逐次勾配降下（各ステップで勾配計算・評価）。
    *   *却下理由*: 各ステップで勾配（backward）を $N$ 回消費するため、TG-LoRA の存在意義である backward 削減（計算コスト削減）が原理的に成立しなくなるため。
*   **却下した代替案2**:
    *   prior 方向の直接適用（フィットなし）。
    *   *却下理由*: 個別サイクルの局所的な曲率や残差成分を無視して外挿を適用することになり、外挿精度が低下しパラメータ崩壊を招きやすくなるため（既存の FD フィット最適化経路と整合）。

## 4. 固定係数化・履歴全捨て純粋外挿方式への移行 (2026-06-07)

### 4.1 意思決定の要約

*   **採用**: FD フィットを固定係数 (alpha=1, beta1=0, beta2=0) に置き換え、accept 後に履歴全捨て＋再ウォームアップする純粋外挿方式。
*   **却下した代替案**: (A) FD フィットの安定化（fp32 化・バッチ拡大・クリッピング）。効果はあるが、v0 の素の有効性を分離して測れないため却下。先に純粋版で v0 を診断する。
*   **位置づけ**: 診断実験。FD フィット恒久廃止の確定ではない。

### 4.2 判断の根拠

*   前回 run で alpha ∈ [-31.66, +7.17], std=4.46 と極めて不安定。
*   accept 3 件はいずれも alpha ≒ 0（v0 寄与ほぼゼロ）で、2/3 が loss 悪化。
*   FD フィットのノイズが設計意図（$N$ 倍 v0 外挿）を完全に遮断している状態で、FD を改善する前に「v0 外挿自体が有効か」を確認する必要がある。
