## 背景（replay は nested sub-verdict CI floats を artifact-rederived 真値に bind しない・TASK-0171 が label のみ bind して floats を残置）

`scripts/replay_freeze_validloss_ci.py` は任意の committed deposit を GPU-free で再判定する chokepoint である。producer（`scripts/run_freeze_validloss_ci_9b._baseline_ci_to_json`/`_direction_ci_to_json`）は §4 sub-verdict を nested slot に stamp する：

```
"baseline": {                                          # §4 条件(a) candidate vs full-backprop
  "verdict": baseline_ci.significance_verdict,         # STRING (label): TASK-0171 bind 済
  "candidate_mean": baseline_ci.candidate_mean,        # FLOAT: 未 bind
  "baseline_mean": baseline_ci.surrogate_mean,         # FLOAT: 未 bind
  "point_improvement": baseline_ci.point_improvement,  # FLOAT: 未 bind (cited magnitude)
  "lower": baseline_ci.lower, "upper": baseline_ci.upper,  # FLOAT: 未 bind (cited CI bounds)
  "confidence": ..., "is_thin_evidence": ...,          # 未 bind
  "n_candidate": ..., "n_baseline": ...                # 未 bind
}
```

TASK-0171（`371e934`）は `_subverdict_rederived`/`_subverdict_stale` を新設して sub-verdict **label 文字列**を per-arm losses から再導出して bind した。しかし：

- `_subverdict_rederived` は `surrogate_valid_loss_ci(...)` の full CI object を構築して **`.significance_verdict` のみ**を返し、lower/upper/point_improvement/means を**破棄**する
- `_ci_stats_stale`/`6ed0f69` は `_CI_STAT_BINDINGS` を `data[stored_key]` で走査し **top-level のみ**を読み、nested `baseline.*`/`direction.*` には**一切降りない**

ゆえに replay の検査は：

- `replay_samples` が main verdict を再導出し `faithful` で照合 ✓（**main verdict STRING のみ**）
- `_ci_stats_stale`/`6ed0f69` が **top-level** CI floats を再導出 ✓（**nested sub-verdict floats は覆わない**）
- `_subverdict_stale`/`371e934` が sub-verdict **label 文字列**を再導出 ✓（**label のみ・floats は破棄**）
- `evidence_hash`/ledger が raw evidence を照合 ✓（**derived stats は evidence ではない**）

の一方で、**nested sub-verdict の cited CI floats（point_improvement/lower/upper/means）を artifact-rederived 真値と照合する検査は一つもなかった**。

ゆえに**内部的に完全に整合した** hand-edited deposit：

- stored main `verdict`（STRING）は losses と一致（`faithful:True`）
- stored `baseline.verdict`（label）は `baseline_losses` から再導出した verdict と一致（`baseline_verdict_stale:False`）
- top-level CI floats は losses と一致（`ci_stats_stale:[]`）
- `evidence_hash`/ledger 全 gate green

が `baseline.point_improvement` だけを 0.18→0.95 に、`baseline.upper` を 0.20→0.99 に過大主張（label は SURPASSES のまま正直）しても、stale field も note も**存在せず完全 silent** に通る。これは corrupt-but-green §4 の**§4 条件(a) magnitude claim** を成立させる silent contract-break（GOAL §7）— TASK-0171 が label のみ bind して残した nested 残存経路。

## 設計（nested-sub-CI-stats binding via own gate・`6ed0f69` の nested 拡張・`_subverdict_ci` primitive・artifact-when-present discipline）

- **`6ed0f69` の nested 拡張**: top-level `_CI_STAT_BINDINGS`/`_ci_stats_stale` と**同一 hazard class**（cited derived statistic trusted over artifact reality）を nested sub-verdict に拡張。新 `_SUBVERDICT_CI_BINDINGS`（slot → `(stored_key, ci_attr)` tuple）は両 slot で candidate/CI-bounds/confidence/thin/sample-size を共有し comparison-arm 名だけ relabel（baseline slot = `baseline_mean`→`surrogate_mean`/`n_baseline`→`n_surrogate`・direction slot = `control_mean`→`surrogate_mean`/`n_control`→`n_surrogate`）= producer の `_baseline_ci_to_json`/`_direction_ci_to_json` 契約に忠実。
- **`_subverdict_ci` primitive**: 旧 `_subverdict_rederived`（`.significance_verdict` のみ返し CI object を破棄）を、full `SurrogateValidLossCI` object を返す primitive `_subverdict_ci(data, *, losses_key)` に refactor。`_subverdict_rederived` は `_subverdict_ci(...).significance_verdict` を返す thin wrapper に縮退（TASK-0171 の label gate と新 float gate が**同一 deterministic bootstrap を 1 回だけ**再利用 = 再計算の重複なし・seed 契約不変）。
- **artifact-rederived 真値**: nested CI は `surrogate_valid_loss_ci(candidate_losses, arm_losses, seed=base_seed)` で再導出（TASK-0171 と**同一 source・同一 seed**）。arm 未実行時 None → skip。
- **producer 側変更不要**: nested floats は既に stamp 済・seed 契約は TASK-0171 と同一 = replay-gate-only fix。
- **artifact-when-present-else-skip**: deposit が該当 slot を持たない（arm 未実行・`None`）・CI を再導出できない・該当 stat を stamp しない場合は skip（`[]`）= 他 gate と同一 posture。legacy/proxy deposit で false-positive しない。
- **単一 helper・再利用**: `format_replay`（prose `DIRECTION_CI_STATS_STALE`/`BASELINE_CI_STATS_STALE` note）と `replay_to_json`（machine field `direction_ci_stats_stale`/`baseline_ci_stats_stale`）の**両方**が同一 `_subverdict_ci_stats_stale` を呼ぶ = §6.2 prose/JSON drift 不能（`6ed0f69`/`371e934`/`47226b9`/`79577a5`/`1ed2def`/`3e3aca6` と同じ構造）。
- **list は bare key**: machine field `baseline_ci_stats_stale` は slot 名を field 名に含むので list item は bare key（`["point_improvement","upper"]`）= top-level `ci_stats_stale` と同一 format。
- **residual vector の誠実記載**: nested float 単独反転（demonstrated reachable corruption）は fail-loud。`baseline_losses` 自体を同時反転する coordinated flip は別 gate が検知（`_ledger_losses_stale`/`_evidence_hash_stale`/`_subverdict_stale`〔label〕が覆う）= 他 sibling gate と同一 honesty stance。`citable_as_full_section4_verdict` は nested float 反転時に不変（proof of need で実証）= magnitude 偽装は citability に波及しない（誠実記載）。
