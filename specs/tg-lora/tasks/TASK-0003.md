## タスク概要

TG-LoRA 学習の全パラメータを一元管理する `TGLoraConfig` dataclass を実装する。YAML/JSON からの読み込み、バリデーション、デフォルト値の提供を行い、学習スクリプト（TASK-0004）とチェックポイント（TASK-0005）の基盤となる。

## 主要実装項目

### TGLoraConfig dataclass 定義 🟡

**信頼性**: 🟡 *既存コンポーネントパラメータの統合*

`tg_lora/config.py` に以下のパラメータグループを定義する:

- **モデル設定**: model_name, adapter_rank, adapter_alpha, target_modules
- **学習設定**: lr, batch_size, grad_accum, max_steps, warmup_steps
- **TG-LoRA設定**: K_initial, N_initial, alpha_initial, beta_initial, relative_update_cap, rollback_tolerance
- **層選択**: active_layer_strategy, trainable_lora_scope
- **出力設定**: output_dir, log_every, eval_every, save_every

**実装ファイル**: `tg_lora/config.py`

### YAML/JSON シリアライズ 🔵

**信頼性**: 🔵 *dataclass + PyYAML パターン*

`from_yaml(path)` / `from_json(path)` / `save_yaml(path)` / `save_json(path)` メソッド。PyYAML は optional 依存関係とする。

**実装ファイル**: `tg_lora/config.py`

### バリデーション 🔵

**信頼性**: 🔵 *既存モジュールのバリデーションパターンに準拠*

`__post_init__` でパラメータ範囲を検証。`RandomWalkController` や `Velocity` と同じ ValueError パターン。

**実装ファイル**: `tg_lora/config.py`
