## 背景（char-F1 の degenerate 入力が silent 0.0 として平均に混入し collapse/壊参照を隠す）

`scripts/eval_downstream.py` は README Quick Start の `make eval` 下流評価を担う CUDA 本線スクリプトで、日本語能力 / JSON 書式 compliance を char-level F1（`compute_char_f1`）で評価する。`compute_char_f1(ref, gen)` は ref/gen を空白除去し、either side が空なら `0.0` を返す：

```python
def compute_char_f1(ref, gen):
    ref_clean = "".join(ref.split())
    gen_clean = "".join(gen.split())
    if not ref_clean or not gen_clean:
        return 0.0          # <-- F1 is UNDEFINED here, not genuinely zero
    ...
```

この `0.0` は、真の total non-match（例: `"abc"` vs `"xyz"`）の `0.0` と**区別不可能**。`evaluate_dataset` はこれを `total_char_f1` に足し込み `mean_char_f1 = total_char_f1 / total` で平均するため、モデルが空文字列を生成（collapse / 生成 error）しても dataset の reference が空/欠損（壊参照）でも、その失敗が「Mean Char-F1: 0.5」のようなもっともらしい平均値の中に**黙って隠れる**。これが feedback 命名の「F1 at zero match」silent path であり、`d9ca7f5` が G3 gate で閉鎖した silent-drop class の downstream 評価における同型である。

`d9ca7f5` の閉鎖パターン：score できない task を黙って drop する代わりに `compared_tasks`（survivor）/`dropped_tasks`/`incomplete` で surface し、threshold verdict は survivor 上で走らせつつ drop を名指し + `main()` の INCOMPLETE banner で告知。

## fix（compared-vs-dropped 規律の downstream char-F1 適用）

1. **`is_char_f1_degenerate_input(ref, gen)` pure predicate 新設** — 空白除去後 either side が空（= F1 未定義）を `True`。`compute_char_f1` と cleaning を `_clean_whitespace` 共有 helper に統一（出力 byte-identical）。
2. **`aggregate_char_f1(per_item)` pure 集約器新設** — headline `mean_char_f1` を **comparable 項目のみ**（`degenerate` flag False）で算出し、degenerate 入力を `degenerate_inputs` count + `comparable` count + `incomplete` flag で surface。comparable 0 件なら mean `0.0` + `incomplete=True`（total collapse を clean mean と偽装しない）。
3. **`evaluate_dataset`**: per-item dict に `degenerate` flag を付与し、summary 構築を inline 平均から `aggregate_char_f1(eval_results)` 経由に変更。`total_char_f1` 局所変数は削除。JSON validity/key compliance rate は `total` ベースのまま（空生成 = 不正 JSON = 正しい失敗扱いで silent でないため）。
4. **markdown report**: `## 1. Summary Metrics` 表の下に degenerate > 0 の task ごとに `[INCOMPLETE -- <task>]` note（N/total items degenerate・Mean は comparable のみ）。
5. **`main()` stderr banner**: `d9ca7f5` の INCOMPLETE banner と同形で、degenerate 入力があれば `INCOMPLETE: N downstream item(s) had degenerate inputs ...` を stderr 出力。

`compute_char_f1` 自体の空入力 `0.0` 返却 semantics は既有 test pin（`test_compute_char_f1`）どおり不変 — per-item `char_f1` は透明性のためそのまま保持し、集約時のみ `degenerate` flag で除外する。
