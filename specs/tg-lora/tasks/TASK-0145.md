## タスク概要

TASK-0144 は gate 評価器 `evaluate_paper_gates.py` に「未測定 ≠ 反証」の第3状態を導入し、自らのレポート内で FAIL と区別した。しかし TASK-0144 自身が**後続タスクとして明記**していた（TASK-0144 §依存タスク）: 下流の `consolidate_paper_results.py` は gate ごとに `r.get("passed")` しか読まず、insufficient gate を「not passed」と同一視し続ける。これは「gate 評価器本体の誠実性が今回の価値」として明示的に scope 外に置かれた残り作業である。

結果として、未測定 gate を含む run を `consolidate_paper_results.py` に通すと、以下の静かな不整合が起きていた:
- **Gate Overview 表**: insufficient gate が `FAIL` と表示される（反証と読める）。
- **`gate_summary`**: `failed` に insufficient が混入し「反証された gate 数」を水増しする。
- **claim-level JSON**: どの claim が「反証で潰れた」のか「まだ測っていない」のか判別不能。
- **Claim Ladder**: 次の claim に届かない理由が「反証」一色に潰れる。

本タスクはこれを**検証可能な振る舞い差分**として閉じる（doc-only ではない）。`evaluated` flag を末端まで伝播し、未測定 gate を `INSUFFICIENT`（人間可読）/ `"insufficient"`（機械可読）で区別する。同時に、claim 報告に「次 claim は**反証で阻まれている**のか**未測定で保留**なのか」を明示する `claim_blockers` を新設する（claim ladder は1段階ごとに1 gate 増える構造なので、次 claim の阻害要因は常にその1 gate）。これは「未測定 ≠ 反証」という GOAL.md §7 の科学的誠実性鉄則を、claim-level 報告の末端まで拡張したものである。

## 設計の拘束条件（後方互換）

- **全 gate が evaluated（TASK-0144 以前の歴史的 run）**: `_gate_outcome` は pass/fail に潰れるため、Gate Overview・`gate_summary.failed`・markdown は**全て byte-identical**。`gate_summary.disproven == failed`・`insufficient == 0`。
- **`passed` / `failed` キーは保存**: `gate_summary` の既存キー（`total`/`passed`/`failed`）は意味・値ともに不変。`disproven` / `insufficient` / gate エントリの `evaluated`+`status` は**追加キー**のみ。
- **`determine_claim_level` は不変**: 達成 claim は引き続き passed gate から計算（既存6テスト緑）。誠実性のnuanceは `claim_blockers` が担い、達成値の意味は変えない。
