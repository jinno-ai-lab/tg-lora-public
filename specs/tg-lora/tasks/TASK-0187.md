## 背景（replay は候補腕の train-valid gap 浮動小数点を artifact 再導出値に bind しない・regime 離散化が精密値を捨てて残置）

`scripts/replay_freeze_validloss_ci.py` は任意の committed deposit を GPU-free で再判定する chokepoint である。producer の四軸 gate の regime 軸（`classify_regime`）は候補腕の train-valid gap を 0.5 で閾値化して離散化する：

```
regime = classify_regime(candidate_final_ce_train_loss_mean, candidate_mean)
       # 内部: if ce < 0.5: MEMORIZATION; elif (candidate_mean - ce) > 0.5: OVERFIT; else GENERALIZATION

candidate_train_valid_gap = candidate_mean - candidate_final_ce_train_loss_mean   # ← 本 TASK が bind する精密 float
```

`_regime_label_stale`（`fd8c370`）は**離散 label** を `classify_regime(ce, candidate_mean)` から再導出して bind する。しかし：

- `_regime_label_stale` は閾値化（離散化）するため**精密 gap 値を捨てる**。同一 regime 内での gap 塗替え（例: 真の +0.01 generalization gap を +0.49 に塗替える — どちらも 0.5 未満 → regime は 'generalization' のまま）は `regime_label_stale=False` で通過する。
- `_ci_stats_stale`（`6ed0f69`）の 9 統計は loss 派生 CI statistic のみで、train-CE 派生の gap を意図的に含まない。
- `evidence_hash`（`79577a5`）は derived statistic を意図的に除外する（gap は非 evidence の派生値 `candidate_final_ce_train_loss_mean` を引く）。

ゆえに citable な full deposit の gap +0.329 を +0.01 に塗替える嘘は、regime も `citable_as_full_section4_verdict` も不变（両方とも artifact 派生を使う）→ `regime_label_stale=False`・`citation_label_stale=False`・`faithful=True`・`ci_stats_stale=[]`・`evidence_hash_stale=False`・`citable_as_full_section4_verdict=True` のまま全 gate green で通る（proof of need: `test_hand_edited_gap_underclaim_is_flagged` が実証）。

## 実装（replay-gate-only）

`_train_valid_gap_stale(data, ci)` helper 新設（torch-free・`freeze_verdict_honesty` と同じ artifact-when-present-else-skip 規律）：

```python
stored = data.get("candidate_train_valid_gap")
ce = data.get("candidate_final_ce_train_loss_mean")
if stored is None or ce is None:
    return False  # artifact-when-present-else-skip
rederived = ci.candidate_mean - float(ce)
return stored != rederived
```

- `ci.candidate_mean` は既存 `_ci_stats_stale` で byte-exact 再導出一致を検証済（決定論 bootstrap）。
- stamped `candidate_final_ce_train_loss_mean` は読み戻すだけ。
- ゆえに `ci.candidate_mean - stamped_ce` は producer の `candidate_valid - candidate_ce_mean` と bit-identical（IEEE754 減算は決定論的）= 全 committed deposit で false-positive なし。
- `TRAIN_VALID_GAP_STALE` prose note（`format_replay`）+ `train_valid_gap_stale` machine field（`replay_to_json` 結果 dict）= prose==JSON 同一契約（SYSTEM_CONSTITUTION Rule #3 / §6.2）。
