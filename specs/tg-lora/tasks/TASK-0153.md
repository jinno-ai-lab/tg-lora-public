## 注意事項

- **axis-multiplication でない**: 本変更は torch-artifact の load path のうち torn 診断を持たない **最後の 1 つ**（`load_trajectory_delta_artifact`）を閉じるものであり、新しい集計軸を増やすものではない（feedback bullet 2/3 が批判する "5th symmetric aggregation axis" は別物＝AI Hub summary の display-only metric 族で、本 mirror には存在しないことを grep 確認済）。consumer は 8+ の offline 解析 script であり、不透明 crash → actionable fail-loud への変換という実効値を持つ。
- **`optimizer.pt` を fabricate しない**: feedback が名指しした分離 `optimizer.pt` は private repo 由来の語彙。本 mirror では baseline optimizer は既に原子化+診断済の `training_state.pt` blob 内にあり、TG-LoRA は optimizer state を永続化しない（設計）。存在しない file を作ることは honesty 違反。
- **upstream sync**: `checkpoint_integrity.py` は mirror-only 新規 leaf（`atomic_save.py` と同様・次回 sync で保持）。upstream（private）には別の `_atomic_write` / 整合性 infra が存在する可能性があるが、本 mirror の primitives はそれと同名・同義を主張しない（`_atomic_torch_save` と同様の命名分離）。
