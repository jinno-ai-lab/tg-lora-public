学習中のサイクル健全性を監視し、発散・停滞・異常パターンを検出するモジュール。JSON形式の健全性レポートを出力し、介入推奨を生成する。

## 主要実装項目

### 1. サイクル健全性モニター (src/tg_lora/cycle_monitor.py) 🟡

**信頼性**: 🟡 *velocity.py/delta_tracker.pyの監視拡張として推測*

- `CycleMonitor.__init__(patience: int = 5, spike_threshold: float = 2.0)`: 初期化
- `CycleMonitor.update(cycle_data: dict) -> HealthReport`: サイクルデータ記録・健全性評価
- `CycleMonitor.detect_divergence() -> DivergenceReport | None`: 発散検出
- `CycleMonitor.detect_stagnation() -> StagnationReport | None`: 停滞検出
- `CycleMonitor.recommend_intervention(report: HealthReport) -> list[str]`: 介入推奨生成
- `CycleMonitor.health_summary() -> dict`: JSON健全性サマリ

**実装ファイル**: `src/tg_lora/cycle_monitor.py`

### 2. データ構造 🟡

**信頼性**: 🟡 *健全性監視のデータモデル設計*

- `HealthReport` dataclass: status, divergence, stagnation, recommendations, cycle_count
- `DivergenceReport` dataclass: detected, metric, severity, current_value, threshold
- `StagnationReport` dataclass: detected, cycles_without_improvement, best_value, current_value

**実装ファイル**: `src/tg_lora/cycle_monitor.py`

## 基本的なテスト要件

### 発散検出テスト 🔵

**信頼性**: 🔵 *発散検出の数学的定義より*

- loss急増（前回の2倍）で発散検出
- NaN lossで発散検出
- 正常パターンで発散なし

### 停滞検出テスト 🔵

**信頼性**: 🔵 *patience-based停滞の定義より*

- patience以上の無改善で停滞検出
- 改善継続時は停滞なし
- 境界値（patience-1）で停滞なし

### 介入推奨テスト 🟡

**信頼性**: 🟡 *介入ロジックは経験則に基づく*

- 発散時: lr低減・K増加を推奨
- 停滞時: lr増加または再開を推奨
