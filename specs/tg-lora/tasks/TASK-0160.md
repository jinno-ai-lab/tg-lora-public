## 背景（feedback が指した「独立再現性」の未達半分）

TASK-0159（iters 22-23）で evidence content-hash と run-log artifact infra を着地したが、**repo の最も citable な結果 = full-budget §4 verdict（`citable_as_full_section4_verdict=True`・`4b88ca8`）は committed byte のみから独立再現可能ではなかった**: run の resumability ledger は gitignored `runs/` 配下（実 run は private stable dir `/home/jinno/tg-lora-public-full-run/` から発火）にあり、skeptic は committed JSON を読んでも記録 loss を独立に再構築できなかった。evidence-hash は「committed byte が不変・監査可能」を担保するが「GPU が byte を生成した」は担保しない（TASK-0159 OPEN #1 と同形）。

## 実装（`7489023`）

- **`tests/fixtures/freeze_validloss_ci_9b_full_ledger.jsonl`（NEW committed witness）**: run の ledger の byte-identical copy — 10 JSONL 行（1 config-fingerprint header + 9 arms: candidate×3 / surrogate×3 / baseline×3）。各 arm は EXACT な `valid_loss` + provenance（`frozen_layers` / `n_trainable_params` / `last_train_loss` / `final_ce_train_loss`）を carry。
- **`tests/fixtures/freeze_validloss_ci_9b_full.json`（additive stamp）**: `ledger_witness_path`（relative repo-root path）+ `ledger_witness_sha256`（parsed lines の canonical SHA-256・`sort_keys`+compact separators・`_run_log_sha256`/`_evidence_hash` と同 discipline）を外科的に追加。**runtime `ledger_path` は `null` のまま**（witness = committed VERIFICATION copy・fresh `make` run が書く incrementally-written resumability path とは区別）。**`evidence_hash` は不変**（witness は provenance であり evidence ではない → `EVIDENCE_HASH_KEYS` に非含）。

## 受け入れ test（`tests/test_run_freeze_validloss_ci_9b.py`・`TestCommittedLedgerWitness` +7・mutation 証明済）

- `test_witness_file_is_committed_and_deposit_points_at_it` — witness は committed repo file・`ledger_witness_path` は relative（absolute worktree-specific path を禁止）。
- `test_witness_hash_is_self_consistent` — stamp == committed byte からの recompute（stale/hand-edit/truncated hash 検出）。
- `test_witness_hash_is_pinned_to_frozen_literal` — stamp == `dfdf22f4…5db86aa`（凍結 literal・**coordinated repaint が通せない唯一の test**・truncated hash も self-consistency は素通りするが凍結 literal ではじく）。
- `test_ledger_reconstructs_deposit_loss_vectors_exactly` — **独立再現性 link**: 全 ledger arm（role+index）が deposit の `candidate_losses`/`surrogate_losses`/`baseline_losses` を float `==` で EXACT 再構築。
- `test_ledger_header_fingerprint_matches_deposit_run_config` — header fingerprint（resumability config-gate）が deposit run config と一致（stale carry-over 検出）。
- `test_runtime_ledger_path_stays_null_for_one_shot_harvest` — semantic guard: `ledger_path`(runtime) は null・`ledger_witness_path`(committed copy) は set（両 field の非混同）。
- `test_witness_is_not_in_evidence_hash_keys` — additivity guard: witness field flip で `evidence_hash` 不変。
- **mutation 証明**: (a) deposit stamp の末尾 hex を perturb ⇒ self-consistency + frozen-literal の 2 つ RED。(b) ledger arm の valid_loss を perturb ⇒ EXACT 再構築 + self-consistency の 2 つ RED。restore で 7 GREEN。
