## 背景（gate が stored boolean のみを信頼し divergent budget を無視する）

`scripts/replay_freeze_validloss_ci.py::replay_to_json` は `citable_as_target_scale` gate を `not proxy_scale and not synthetic and not negative_control` で計算する。ここで `negative_control = bool(data.get("negative_control", False))` — **file に保存された operator-set boolean のみ**を読む。同 boolean は prose note（`format_replay` の NEGATIVE_CONTROL block）の gate でもある。

一方、**producer**（`scripts/run_freeze_validloss_ci.py` lines 619-629）は `negative_control` を per-arm budget divergence（`candidate_total`/`surrogate_total` vs `total`）から**導出**する — degraded arm を作れば必ず flag が立つ。`replay` 側にも `_resolve_negative_control_arm(data)` が存在し、budget から degraded arm を**機械的に特定**できる（`negative_control_arm` field 無しでも `total` との比較で解決）。

しかし gate はこの機械的判定を**使わず** stored boolean のみを信頼する。ゆえに hand-edited / externally-supplied deposit が:

- `candidate_total=2` / `total=60`（degraded candidate arm）
- ただし `negative_control` field 欠落（または explicit `false`）

を運ぶと、gate は `negative_control=False` と読み、`proxy_scale=False`/`synthetic=False` なら **`citable_as_target_scale=True`** を返す。これは degraded-arm sensitivity probe が **citable §4 target-scale order result** に化ける silent corruption である（GOAL §7）。producer が書いた deposit では到達不能（producer は divergence から flag を導出する）だが、replay は**任意の artifact を再判定する chokepoint** であり、producer の不変量に依存してはならない。

到達性: `freeze_validloss_9b_target.json` は `total`/`candidate_total`/`surrogate_total` を持つ（symmetric 15/15/15・flag false）ので、divergence を運ぶ schema は committed corpus に**既に存在**する。同 deposit の budget が symmetric であることだけが現状 gate を守っている — divergence に変われば silent に citable に化ける。

## 設計判断（gate chokepoint での単一 helper・label でなく artifact 優先）

- **helper 位置**: `_negative_control_active(data)` を `_resolve_negative_control_arm` の直後に新設（既存の budget→arm 解決を再利用・重複ロジックなし）。`replay_to_json` の gate と `format_replay` の prose note の**両方**がこれを呼ぶ = §6.2 の「prose と JSON は同一契約を執行し drift できない」を維持。
- **優先順位**: `flag OR divergence`。flag=True なら無条件 True（明示的 provenance を尊重）。flag 欠落/False でも divergence あれば True（artifact reality が label に優先）。**operator は flag=false で gate を defeat できない** — これが本 fix の核心（label trusted over reality の逆転）。
- **producer 不変量との非重複**: producer は divergence→flag を導出するが、replay は producer を経由しない外部 artifact も扱う。両者が一致する committed corpus（後述 invariant test）は byte-identical だが、gate は producer の善意に依存しない独立防衛を持つ = TASK-0152 の「手書き転写を artifact 直読で置換」と同 posture。

## fix

`scripts/replay_freeze_validloss_ci.py`:

1. `_negative_control_active(data)` helper 新設 — `bool(data.get("negative_control", False)) or _resolve_negative_control_arm(data) is not None`。docstring に corruption 機構（stored label が budget reality を上書きする silent contract-break）と TASK-0152/`5ed3380` との同 class 関係を明記。
2. `replay_to_json`: `negative_control = bool(...)` → `_negative_control_active(data)`。gate（`citable_as_target_scale`/`citable_as_full_section4_verdict`）と machine-readable `negative_control` field の両方が effective 値を使う。docstring bullet も更新。
3. `format_replay`: `negative_control_flagged`（stored）と `negative_control`（effective）を分離。scale line は effective 値を表示（reader は gate の実際の挙動を見る）。prose は flagged-vs-unflagged で分岐:
   - **flagged**: 既存 NEGATIVE_CONTROL note（「deliberately degraded」・asserted sensitivity probe）そのまま。
   - **unflagged divergence**: 新規 `BUDGET_DIVERGENCE_UNFLAGGED` note — arms' budgets が diverge するが flag が `False`/unset であること・gate が divergence から citation を withhold すること・`negative_control: true` を set するか budget を直すかの対処を明示。「deliberately」は**言わない**（gate は divergence を検出するが intent は不明）。

保証: **divergent-budget deposit は flag の有無に関わらず citable §4 order result にならない**。producer が書いた committed corpus では flag==divergence なので effective==stored（byte-identical）。gate と prose は同一 helper 経由で drift しない（§6.2）。
