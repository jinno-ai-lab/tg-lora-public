## タスク概要

TG-LoRA 学習の全状態を単一ファイルに保存し、中断箇所から正確に再開できるチェックポイント機能を実装する。各コンポーネントの `summary()` / `from_dict()` を統合し、アトミック書き込みで破損を防止する。

## 依存タスク

- **前提タスク**: [TASK-0003](TASK-0003.md), [TASK-0004](TASK-0004.md)
- **後続タスク**: なし（Phase 2 最終タスク）

## 完了条件

- [ ] `save_checkpoint()` でモデル・オプティマイザ・全 TG-LoRA 状態を保存
- [ ] `load_checkpoint()` で正確に復元し学習を再開
- [ ] アトミック書き込み（temp file + rename）で破損防止
- [ ] チェックポイントファイルにタイムスタンプとメタデータを含む
- [ ] 復元後の reduction_rate・acceptance_rate が連続する

## 主要実装項目

### チェックポイントシリアライズ 🟡

**信頼性**: 🟡 *各コンポーネントの summary()/from_dict() から推測*

保存対象:
- **モデル**: `model.state_dict()` + PEFT adapter weights
- **オプティマイザ**: `optimizer.state_dict()`
- **RandomWalkController**: `controller.state.summary()`
- **CycleState**: `cycle_state.summary()`
- **DeltaTracker**: norms と last_stats
- **Velocity**: 現在の state dict
- **TGLoraConfig**: 設定の完全な記録
- **TrajectoryAnalyzer**: ポイント履歴

**実装ファイル**: `tg_lora/checkpoint.py`

### アトミック書き込み 🔵

**信頼性**: 🔵 *標準的なファイル I/O パターン*

```python
def save_checkpoint(path, state_dict):
    tmp = path.with_suffix('.tmp')
    torch.save(state_dict, tmp)
    tmp.rename(path)  # atomic on POSIX
```

**実装ファイル**: `tg_lora/checkpoint.py`

### 復元と検証 🟡

**信頼性**: 🟡 *各コンポーネントの from_dict/restore_state に基づく*

`load_checkpoint()` で:
1. ファイル存在確認と整合性チェック
2. 各コンポーネントの `from_dict()` / `restore_state()` で復元
3. サイクル番号・ステップ数の連続性確認

**実装ファイル**: `tg_lora/checkpoint.py`
