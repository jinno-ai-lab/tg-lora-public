## 背景（replay は候補腕の mean train-CE 浮動小数点を provenance 再導出値に bind しない・派生サマリが evidence_hash を潜り抜ける）

`scripts/replay_freeze_validloss_ci.py` は任意の committed deposit を GPU-free で再判定する chokepoint である。producer は候補腕の mean train CE を次のように stamp する：

```
candidate_final_ce_train_loss_mean = _candidate_final_ce_mean(result)
    # = mean(float(prov["final_ce_train_loss"]) for prov in candidate_provenance if finite)
candidate_train_valid_gap          = candidate_mean - candidate_final_ce_train_loss_mean   # TASK-0187 bind 済
regime                             = classify_regime(candidate_final_ce_train_loss_mean, candidate_mean)  # TASK-0186 bind 済
```

`candidate_provenance`（各腕 `final_ce_train_loss`）は hash-bound EVIDENCE であり、`_evidence_hash_stale`（`79577a5`）が provenance byte の塗替えを捕まえる。しかし：

- top-level `candidate_final_ce_train_loss_mean` は**派生サマリ**（derived statistic）で、`EVIDENCE_HASH_KEYS` に含まれない（verdict/gate/regime と同じ非 evidence クラス）→ `evidence_hash` は触れない。
- `_regime_label_stale`（`fd8c370`）/`_train_valid_gap_stale`（`2d56d59`）は両方とも ce_mean を**INPUT として読む**だけで、provenance が要約する真値と照合しない。

ゆえに citable な full deposit の ce_mean ~1.366 / gap +0.329 を、ce_mean を ~1.675 に・gap を `candidate_mean - 1.675` ≈ 0.02 に**整合的に**（かつ同一 regime 内で）塗替える嘘は、provenance を触らないため `evidence_hash_stale=False`、gap も整合塗替えのため `train_valid_gap_stale=False`、regime bucket 不变のため `regime_label_stale=False`、`citable_as_full_section4_verdict` も stored ce_mean（in-regime）から再導出で `True` のまま → 全 gate green で通る（proof of need: `test_coordinated_repaint_of_ce_mean_and_gap_is_flagged` が実証）。TASK-0187/0186 docstring の「train-CE artifact と label の協調塗替えは `evidence_hash_stale` が捕まる」という主張は、**provenance を触った場合のみ成立**する — 派生サマリ（非 evidence）だけの塗替えは潜り抜ける。

## 実装（replay-gate-only）

`_candidate_final_ce_mean_from_deposit(data)`（torch-free・producer `_candidate_final_ce_mean` と bit-for-bit 同一算術：`float()`-coerce + `math.isfinite` skip + `sum/len`）+ `_candidate_final_ce_train_loss_mean_stale(data)` helper 新設（`freeze_verdict_honesty` と同じ artifact-when-present-else-skip 規律）：

```python
stored = data.get("candidate_final_ce_train_loss_mean")
if stored is None:
    return False  # artifact-when-present-else-skip
rederived = _candidate_final_ce_mean_from_deposit(data)  # mean over candidate_provenance finite final_ce_train_loss
if rederived is None:
    return False
return stored != rederived
```

- `candidate_provenance` は producer が verbatim で stamp するため、本再導出は producer の `_candidate_final_ce_mean` と bit-identical（IEEE754 加算/除算は決定論的・全 committed deposit で false-positive なし）。
- `_candidate_final_ce_mean_from_deposit` を stale 判定と prose note で共有 = drift 不可。
- `CANDIDATE_FINAL_CE_TRAIN_LOSS_MEAN_STALE` prose note（`format_replay`）+ `candidate_final_ce_train_loss_mean_stale` machine field（`replay_to_json` 結果 dict）= prose==JSON 同一契約（SYSTEM_CONSTITUTION Rule #3 / §6.2）。
