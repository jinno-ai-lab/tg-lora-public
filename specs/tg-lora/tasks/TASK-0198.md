## 背景（なぜ「再計算でも prose でもなく、決定の着地機構」か）

feedback（value-judge）は TASK-0197 を VALUABLE 評価し、verdict arc が完了したことを認めた上で、**次 iter の最高レバレッジ**を明示した:

> The single highest-leverage move is to convert the documented accept-null
> decision into either (a) a landed default flip gated behind an explicit
> operator-approved flag/commit, or (b) a single blocking operator prompt
> requesting the ship/accept-null/pivot call — then stop surfacing variants
> until that call lands.

同時に feedback は**禁止事項**を明示: (1) re-derivation/verdict-scaffolding tooling 追加＝R3 churn・(2) third profile 追加・(3) README/decision prose 追加＝「peripheral-plumbing lane」。

検証の結果:

1. **accept-null/ship/pivot 決定は実在し文書化済み**: TASK-0195 が `section4_operator_decision.py` で決定を machine-verifiable 化（`recommendation=SHIP`・3 分岐・`arc_complete=true`）。PURPOSE.md L1232-1233 にも文書化済み。**しかし着地機構が無い**＝決定は「推奨」として emit されるだけで、binding な operator 呼び出しとして record されない。これが「文書化→着地」変換の未達部分。
2. **feedback 命名の phantom lever は本 mirror に不在（grep 実測）**: feedback は「the `single-pass` escape hatch exists and the efficacy evidence is locked (τ=+0.522; refinement net-negative at every scale)」と前提するが、`rg 'single-pass|single_pass|0\.522|kendall|max_passes|recursive\.enabled' src/ scripts/ tests/` = **0 hit**（`tau` hit は全て `dynfreeze_tau`=0.015 の freeze 閾値で Kendall 順位相関 τ と無関係）。TASK-0167〜0177 と同結論＝これらは AI-Hub-private 記号（efficacy arc は AI Hub 自身の runner、または private repo に存在）。ゆえに phantom には触れない。
3. **「一方的に決めない」は誠実性 core（§7）**: TIES（null）結果に対する ship/accept-null/pivot は研究者の人間判断であり、ツールが一方的に決めない（feedback も "a call you are correctly not making unilaterally" と明記）。

ゆえに: verdict 再計算（禁止）・prose（禁止）・phantom lever（不在）をすべて排除し、**実在する決定の着地機構**のみを実装する。着地は operator が行い（`--land`）、ツールは着地されるまで blocking し、着地されると variant を出さず停止する。これが (a)〔`--land` で書かれる committed record = operator 承認 flag/commit〕と (b)〔未着地時の単一 blocking prompt〕の融合。

## Decision trace

- **phantom lever 拒否**: feedback の「single-pass escape hatch / τ=+0.522 efficacy」は grep で本 mirror 不在（TASK-0167〜0177 系譜）。efficacy arc（TASK-0118→0145）は feedback 自身が COMPLETE 宣言・PURPOSE も追加計測禁止。これらに触れると phantom を捏造することになるため却下。
- **verdict 再計算・scaffolding 拒否**: feedback が R3 churn と明示。本変更は verdict を再計算せず・metric に guard を追加せず・既存 replay primitive をそのまま使用（`arc_complete` は deposit-derived のまま）。
- **prose 拒否・着地機構採用**: README/decision prose 追加は feedback 禁止 lane。代わりに「文書化された決定」を「着地可能な blocking 表面」へ変換＝feedback 命名の highest-leverage move。(a)〔`--land` committed record = operator 承認 flag/commit〕と (b)〔単一 blocking prompt〕を融合: record が operator 承認 commit であり、未着地時の blocking prompt が (b)。
- **「一方的に決めない」の保持**: ツールは SHIP/ACCEPT-NULL/PIVOT を決定しない。operator が `--land` で着地するまで blocking し、着地後も evidence-based recommendation（SHIP）と operator の landed 呼び出しを分離して両方表示する（`test_land_does_not_mutate_recommendation` で pin）。
- **CI 非破壊**: `section4-operator-decision` は `ci:` target に含まれない（standalone）ため、exit 3（blocking）化しても CI は green のまま。operator が手動起動した時のみ blocking が見える＝意図通りの forcing function。
