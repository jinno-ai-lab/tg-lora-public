## 背景（空 corpus が silent clean 0.0 に偽装する経路）

TASK-0190 は `aggregate_char_f1` を新設し、degenerate 入力（空 ref/gen = F1 未定義）の silent `0.0` が `mean_char_f1` の平均に混入する silent path を `incomplete` flag で surface した。しかし `incomplete = bool(degenerate_inputs)` は**空 corpus**（`per_item == []`、即ち `total == 0`・`comparable_count == 0`・`degenerate_inputs == 0`）を取りこぼし、`incomplete=False`・`mean_char_f1=0.0` を返す。これは TASK-0190 point 2 が「comparable 0 件なら `incomplete=True`（total collapse を clean mean と偽装しない）」と宣言したルールに違反する（空 corpus は comparable==0 だから）。

結果：空/破損 dataset（`load_jsonl` が `[]` を返す等）や全 prompt 空で `evaluate_dataset` が 1 件も評価しないとき、report は「Mean Char-F1: 0.0000」を**INCOMPLETE 無しで**出力し、stderr banner も発火しない。読者は 0.0000 を「全問 zero match（破滅的失敗）」と「0 件評価（評価未実行）」から区別できず、まさに TASK-0190 が閉じようとした silent path の空 corpus 同型が残っていた。test `test_aggregate_char_f1_empty_corpus` は `incomplete is False` でこの挙動を pin していたが、これは point 2 設計意図に反するバグの pin（deliberate choice を示す comment なし）。

## fix（empty corpus を degenerate と同型に surface）

1. **`aggregate_char_f1`**: `incomplete` を `bool(degenerate_inputs) or total == 0` に拡張。空 corpus（zero-comparable）は mean が情報を持たないため incomplete。docstring に空 corpus の扱いと point 2 整合を明記。`mean` 算出（`comparable_count else 0.0`）・他 field は不変。
2. **markdown report INCOMPLETE note**: 各 task で `summary["total"] == 0` なら `[INCOMPLETE -- {label}]: 0 items evaluated (empty corpus); Mean Char-F1 is undefined.` を書き `continue`（degenerate note と排他）。degenerate ブランチは既存のまま。
3. **stderr banner**: 既存の degenerate banner に加え、空 corpus task を集め `INCOMPLETE: 0 items evaluated (empty corpus) for {tasks}; Mean Char-F1 is undefined.` を stderr 出力。
4. **test**: `test_aggregate_char_f1_empty_corpus` の `incomplete` 期待値を `False`→`True` に反転し、point 2 設計意図への整合を comment で明記。

`compute_char_f1` / `is_char_f1_degenerate_input` / degenerate surface は不変（byte-identical）。
