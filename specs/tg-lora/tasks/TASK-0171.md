## 背景（gate は nested sub-verdict label を一切再判定しない）

`scripts/run_freeze_validloss_ci_9b.py::run_ci_9b` は GOAL §4 の **3 つの verdict** を計算する（`run_freeze_validloss_ci_9b.py` lines ~1707-1730）:

1. **main verdict** — `ci = surrogate_valid_loss_ci(candidate_losses, surrogate_losses, seed=base_seed)`（候補 vs ランダム順 surrogate）
2. **direction-isolation sub-verdict** — `direction_ci = surrogate_valid_loss_ci(candidate_losses, control_losses, seed=base_seed)`（§4 P0 confound 隔離・output-contiguous vs input-contiguous）
3. **full-backprop baseline sub-verdict** — `baseline_ci = surrogate_valid_loss_ci(candidate_losses, baseline_losses, seed=base_seed)`（§4 line 247 条件(a)・progressive-freeze 候補 vs no-freeze full-CE）

これらは `result_to_json` → `_direction_ci_to_json`/`_baseline_ci_to_json` で deposit に `{"verdict": ...}` として stamp され、**同時に** `candidate_losses`/`control_losses`/`baseline_losses` も格納される = sub-verdict は **GPU-free で再導出可能**（同じ deterministic bootstrap・同じ seed）。

一方 `scripts/replay_freeze_validloss_ci.py` は:

- **main verdict** は `replay_samples` が `surrogate_valid_loss_ci(candidate_losses, surrogate_losses, seed=base_seed)` で再計算し `faithful` で stored `verdict` と照合する ✓
- **top-level citation boolean** `citable_as_full_section4_verdict` は `d734327` が 4 軸 artifact から再導出し `citation_label_stale` で cross-check する ✓
- **direction/baseline sub-verdict** — `replay_samples` も `replay_to_json` も `format_replay` も**一切再導出しない**（`control_losses`/`baseline_losses`/`direction`/`baseline` への機能的参照 = grep 0）。gate は nested label を黙って信頼する。

ゆえに hand-edited / externally-supplied deposit が `baseline.verdict = "SURPASSES"` を運ぶが `baseline_losses` が実際は `TIES` に計算される場合:

- `faithful` は **True**（main `verdict` は stored と一致・嘘は nested label のみ）
- `citation_label_stale` は **False**（top-level boolean は honest）
- stale field も prose note も**存在しない**

= §4 条件(a) の sub-verdict が嘘をついても **完全に silent** に通る。これは corrupt-but-green §4 を成立させる silent contract-break（GOAL §7）。producer が書いた deposit では到達不能（producer は losses から sub-CI を計算する）だが、replay は**任意の artifact を再判定する chokepoint** であり、producer の不変量に依存してはならない。

到達性（実証）: committed `freeze_validloss_ci_9b_full.json` は `baseline.verdict=SURPASSES` + `baseline_losses`（len=3）を持ち、真の再導出も SURPASSES。この deposit の `baseline.verdict` を `UNDERSHOOTS` に書き換えた実験で、fix 前 replay は `faithful:True`/`citation_label_stale:False`/stale field なし/prose note なし = **完全 silent** を返すことを確認済（proof of need）。

## 設計判断（gate chokepoint での単一 helper・top-level と同じ posture で nested に延長）

- **helper 位置**: `_subverdict_rederived(data, *, losses_key)` と `_subverdict_stale(data, *, slot, losses_key)` を `_citation_label_stale` の直後に新設。`_direction_ci_to_json`/`_baseline_ci_to_json` が candidate-vs-arm の汎用 bootstrap を再利用するのと同様、stale check も slot(`direction`/`baseline`) × losses(`control_losses`/`baseline_losses`) の対で単一 helper を参数化（重複ロジックなし）。`replay_to_json` の 4 field と `format_replay` の 2 prose note の**両方**がこれを呼ぶ = §6.2 の「prose と JSON は同一契約を執行し drift できない」を維持（`d734327`/`9dff092`/`bbf6e68` と同じ構造）。
- **seed 契約**: `seed = int(data.get("base_seed", 0))` — producer が sub-CI を `seed=base_seed` で計算した（lines ~1715-1727）のと同一。これにより committed deposit は bit-for-bit 再導出される（byte-identical 保証）。
- **skip 動作**: arm が未実行（slot が `null`・`control_losses`/`baseline_losses` が absent/empty）なら `rederived=None`/`stale=False`/note なし。committed corpus では `_full.json`（direction=null）・`_direction.json`（baseline=null）・`_surrogate.json`（両方 null）等、arm を走らせなかった deposit が正当に存在し、cross-check はそれらに false-positive を出さない。
- **producer 不変量との非重複**: producer は losses→sub-verdict を計算するが、replay は producer を経由しない外部 artifact も扱う。両者が一致する committed corpus（後述 invariant test）は byte-identical だが、gate は producer の善意に依存しない独立防衛を持つ = TASK-0169 と同 posture。

## fix

`scripts/replay_freeze_validloss_ci.py`:

1. `_subverdict_rederived(data, *, losses_key)` helper 新設 — `candidate_losses` + 指定 arm losses から `surrogate_valid_loss_ci(...).significance_verdict` を `seed=base_seed` で再導出。arm 未実行/losses 欠落時は `None`。docstring に corruption 機構（nested label が per-arm losses reality を上書きする silent contract-break）と `d734327`/TASK-0169 との同 class 関係を明記。
2. `_subverdict_stale(data, *, slot, losses_key)` helper 新設 — slot(`direction`/`baseline`) の `verdict` label と再導出値の不一致を `stored != rederived` で判定。slot absent/null・verdict 欠落・rederived=None の時は `False`。
3. `replay_to_json`: `direction_verdict_rederived`/`direction_verdict_stale`/`baseline_verdict_rederived`/`baseline_verdict_stale` の 4 field を `citation_label_stale` の直後に追加。docstring bullet も更新。
4. `format_replay`: `CITATION_LABEL_STALE` block の直後に `(slot, losses_key, note)` の tuple loop で `DIRECTION_VERDICT_STALE`/`BASELINE_VERDICT_STALE` note を出力（stale 時のみ）。note は stored label・真の再導出値・「artifact 優先」・対処（fresh producer run or label 修正）を明示。

保証: **lying sub-verdict label を持つ deposit は stale=True + rederived=真値 + prose note で fail-loud になる**。producer が書いた committed corpus では rederived==stored なので stale=False（byte-identical）。gate と prose は同一 helper 経由で drift しない（§6.2）。

## test（`TestSubVerdictLabelStaleness` +6・mutation 証明）

`tests/test_replay_freeze_validloss_ci.py` に新設:

1. `test_committed_deposits_sub_verdicts_match_losses` — **byte-identical invariant**: 全 7 committed 9B deposit で、arm が走った slot は `rederived == stored verdict`・両 stale flag False・両 stale note なし。arm 未実行 slot は `rederived=None`/`stale=False`/note なし。将来 diverged fixture が追加されれば trip し silent flip を防ぐ。
2. `test_hand_edited_lying_baseline_verdict_caught_and_overridden` — **primary mutation proof (baseline 軸)**: `_full.json` の `baseline.verdict` を真値と異なる label に書き換え → `baseline_verdict_rederived == 真値`（artifact 勝ち）・`baseline_verdict_stale is True`・`BASELINE_VERDICT_STALE` note・prose に真値が命名。
3. `test_hand_edited_lying_direction_verdict_caught_and_overridden` — **mutation proof (direction 軸・対称)**: `_direction.json` の `direction.verdict` を書き換え → 対称に `DIRECTION_VERDICT_STALE`。両 sub-verdict を cover する（baseline のみでない）。
4. `test_sub_verdict_rederivation_uses_producer_base_seed` — **contract**: 再導出が deposit の `base_seed` を使う（bootstrap default でなく producer が sub-CI を stamp した seed と同一）ことを、直接 `surrogate_valid_loss_ci(..., seed=base_seed)` と照合して pin。
5. `test_sub_verdict_skipped_when_arm_did_not_run` — **false-positive 防御**: `_full.json`（direction=null）は `direction_verdict_rederived is None`/`stale is False`/note なし。`_direction.json`（baseline=null）も対称。正当に arm を走らせなかった deposit に cross-check が false-positive しない。
6. `test_prose_note_presence_matches_machine_stale_flag` — **drift invariant**: 全 committed + 両 mutated case で `<SLOT>_VERDICT_STALE in prose == JSON stale flag`（§6.2・`d734327` の `test_prose_invariant_holds_across_committed_9b_deposits` と同形）。

**mutation 証明**: `_subverdict_stale` の最終 `return stored != rederived` を `return False`（stored-label-trust・fix 前挙動）に戻すと、gate 依存の 2 hand-edited test（#2/#3）が RED（`assert False is True`）。stable 4 test（#1 byte-identical・#4 seed contract・#5 skip・#6 drift-invariant）は GREEN = mutation-stable。復元で全 6 GREEN。hand-edited test が load-bearing であることを確認。
