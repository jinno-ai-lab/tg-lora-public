## operator 決定（表面化する binding constraint）

**相対 §4 verdict arc は COMPLETE**: homogeneous + heterogeneous 両 leg とも citable TIES = TG-LoRA 外挿は full-backprop と**統計的に同等の品質**を full-budget seq1024 で達成（GOAL §0 品質保持を支持）+ backward_passes 14–35% 減でコスト削減の観測的現れ。abort→rerun 起動導線も堅牢化済み。

残る真の open は** private `src.data` absolute-loss 比較（DATA 軸・分類 C・外部依存）**のみ:
- public mirror の verdict runner は upstream real Dolly corpus で絶対 loss を測るが、upstream private baseline の corpus config と完全一致しないため**異 setup 間の絶対 loss 直接比較**が未確立。
- これは code-doable でなく（private `src.data` pipeline の品質 filter 移植が必要・外部依存）・相対 verdict 自体の citability を block しない。

**operator 決定（3択）**:
1. **SHIP**（推奨）: citable TIES 相対 verdict を §4 結果として採用（GOAL §0 品質保持×コスト削減を支持）。DATA 軸絶対比較は optional cross-check として private 側へ deferred・§4 verdict の blocker ではない。
2. **ACCEPT-NULL**: TIES を「TG-LoRA は品質劣化なし・優位もなし」の誠実な null として受理（同結論）。
3. **PIVOT/INVEST**: private `src.data` 品質 filter を public mirror 側へ移植し絶対 loss 比較を確立（別 TDD・外部依存解消が必要・本 checkout からは不可）。

**unblock step（PIVOT を選ぶ場合）**: upstream `/home/jinno/tg-lora/src/data/filter_dataset.py` の品質 gate を public mirror へ移植 → 同一 corpus config で baseline/candidate を再測 → `form_freeze_validloss_deposit` + `freeze-replay` で絶対比較 verdict を記録。abort→rerun 堅牢化済みなので再測中の OOM-before-checkpoint は `make recover ... --rerun` で完走可能。
