## タスク概要

TASK-0146 は `evaluate_paper_gates.py` が bare CLI として起動できず、自身の honesty-contract E2E test 群（13件）を import 段で死なせていた欠陥を、repo-root を `sys.path` に載せる1行 idiom で閉じた。しかしその1行欠陥は **`evaluate_paper_gates.py` だけのものではなかった**: top-level で `from src.* import` する別の6 script が同一の「bare 起動で `ModuleNotFoundError: No module named 'src'` 即死」状態にあり、それぞれの CLI subprocess / E2E test を不活性化していた。

これは TASK-0140〜0146 が一貫して排除してきた「コードは直ったように見えるが実は不活性」という静かな状態の**発生源の残り**である。中でも `benchmark_velocity_ops.py` は **CI 回帰ゲート**（`make bench-velocity-ops-ci`・`TestCIGateBaseline` / `TestBaselineRegressionDetection`）を担うが、この回帰検出網そのものが import 死で完全に不活性だった — velocity/cap_update 操作に回帰が入っても CI は緑のまま気づかない状態だった。

さらに `inspect_model.py` / `compare_runs.py` / `analyze_prefix_cache_break_even.py` / `benchmark_velocity_ops.py` は **Makefile の documented user-facing target** から `$(PYTHON_VENV) scripts/X.py`（PYTHONPATH prefix なし）で bare 起動されており、`make inspect-model` / `make compare-runs` / `make bench-velocity-ops-ci` 等の入口そのものが壊れていた（dedicated test が PYTHONPATH を自前設定していたため test 上は緑に見え、静かに壊れていた）。

本タスクは TASK-0146 と同一の1行 idiom を6 script に機械適用し（`import sys` / `from pathlib import Path` が未 imported の script には併せて追加）、発射性を自己完結的に回復する。新規依存・新規 knob なし。これで (a) 6 script の bare CLI が発射する、(b) 対応 Makefile target が発射する、(c) 20件の死 E2E test が初めて実際の CLI 振る舞いを検証して緑になる。

## 残留1件の正直な取り扱い（`test_ci_gate_passes`）

`benchmark_velocity_ops.py` の `TestCIGateBaseline::test_ci_gate_passes` は本修正で **inert import 死 → 実際の benchmark + 回帰検出ロジックを実行** するようになった（同ファイルの `TestBaselineRegressionDetection::test_baseline_regression_exits_nonzero` は緑化し、回帰検出網が実働したことを証明）。しかし同 test は **環境依存の絶対タイミング閾値** を assert し、この worktree のマシンでは弾かれる:

- checked-in baseline `baselines/velocity_ops.json` は1度だけ（commit `ddc01d4`）別マシンで記録。`velocity_ema_per_iter_ms=0.09ms` 等。
- 本マシンの実測は run-to-run で **ratio 1.6× → 4.3× → 36×**（cap_update は 33〜36×）と激しく変動し、gate 閾値 1.5× を安定して超える。
- これは **bootstrap ともコード欠陥とも無関係の環境的変動**（マイクロ秒以下の演算の絶対タイミング・baseline 記録マシンとの差・マシン負荷）。baseline や閾値を書き換えて緑化すれば、本来の CI マシンでの真の回帰を隠蔽することになるため **意図的に触れない**。本件は public-mirror 既知の「マシン依存・事前から失敗」カテゴリ（GOAL §7 の「評価条件統一」は本件と無関）。
