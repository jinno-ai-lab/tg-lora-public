## タスク概要

MLflowLoggerに一時的なネットワーク障害に対するリトライロジックを追加し、エラーメッセージの詳細化を行う。現在は例外をキャッチしてログ出力するのみで、一時障害時のデータ損失リスクがある。

## 依存タスク

- **前提タスク**: [TASK-0058](TASK-0058.md)
- **後続タスク**: [TASK-0060](TASK-0060.md)

## 完了条件

- [x] MLflow操作に指数バックオフリトライ（最大3回）を追加
- [x] リトライ対象例外（ConnectionError, Timeout）と非リトライ対象を分離
- [x] エラーログに操作種別とメトリクス名を含める
- [x] リトライにより学習ループが停止しないことを確認

## 主要実装項目

### リトライロジック実装 🟡

**信頼性**: 🟡 *MLflow運用パターンから妥当な推測*

tenacityライブラリまたは独自のリトライデコレータで、指数バックオフリトライを実装する。

**実装ファイル**: `src/utils/mlflow_logger.py`

### エラーメッセージ詳細化 🔵

**信頼性**: 🔵 *mlflow_logger.py既存エラーハンドリングより*

エラーログに操作種別（log_metrics, log_artifact等）と対象データ名を含める。

**実装ファイル**: `src/utils/mlflow_logger.py`
