## 背景（replay は §7 materiality boolean を artifact-rederived 真値に bind しない・TASK-0176 が named blocker として残置）

`scripts/replay_freeze_validloss_ci.py` は任意の committed deposit を GPU-free で再判定する chokepoint である。producer（`scripts/run_freeze_validloss_ci_9b.result_to_json`）は verdict deposit に §7 の両軸を stamp する：


TASK-0176（`3e3aca6`）は §7 が**統計的有意性と実用的集中度を区別**すると明記しつつ、統計軸 boolean `significant_surpasses` のみを `ci.significant_surpasses` に bind する gate を新設し、**実用軸** `is_material` については：

> 対照的に `is_material`（`point_improvement >= material_margin`）と `passes` は **margin-dependent**（`material_margin` が deposit に stamp されない）なので bind 対象外

と**明示的に unbound 残置**した（named blocker）。ゆえに replay の検査は：

- `replay_samples` が losses から verdict を再導出し `faithful` で stored verdict と照合 ✓（**verdict STRING のみ**）
- `_ci_stats_stale` / `6ed0f69` が `_CI_STAT_BINDINGS`（lower/upper/means/point/confidence/n/thin = margin-invariant 統計量）を再導出 ✓（**`is_material` は margin-dependent として除外**）

の一方で、**stored `is_material` boolean を artifact-rederived 真値と照合する検査は一つもなかった**。blocker の核心：`is_material = point_improvement >= material_margin` の `point_improvement` は ci から再導出可能（= `_CI_STAT_BINDINGS` に含まれ既に bound）だが、`material_margin` は deposit に stamp されないため replay は producer が用いた margin を知らず、deterministic に再導出できなかった。

ゆえに**内部的に完全に整合した** hand-edited deposit：

- stored `verdict`（STRING）は losses と一致（`faithful:True`）
- stored `lower`/`upper`/means（FLOATS）は ci と一致（`ci_stats_stale:[]`）
- stored `significant_surpasses`（統計 boolean）は `ci.significant_surpasses` と一致（`significant_surpasses_stale:False`）
- `evidence_hash` は stamp と一致（boolean は evidence key ではない）
- 全 citation / ledger / sub-verdict / Level-1/Level-2 boolean gate が green

## 設計（material_margin provenance stamp + ci.point_improvement への binding・TASK-0176 named blocker の解決・artifact-when-present discipline）

- **named blocker の解決**: TASK-0176 が「`material_margin` が deposit に stamp されないため bind 対象外」とした blocker を、producer に `material_margin` を stamp させることで解決する。stamp は provenance-only（caller-set threshold・`confidence`/`n_bootstrap` と同 class・raw measurement ではない）なので `EVIDENCE_HASH_KEYS` に含まれず evidence_hash 不変・`_CI_STAT_BINDINGS` にも含まれず ci_stats 不変。stamp 値は default `0.0`（producer は `material_margin` を override しない）。
- **対をなす軸**: `3e3aca6`（統計軸 boolean `significant_surpasses`）と**対をなす実用軸**（`is_material`）。両者で §7 の「統計的有意性 vs 実用集中度」区別が完全に gate 覆盖される。`passes`（= `significant_surpasses and is_material`）は両 boolean の論理積なので両 gate で transitively に検証され unbound のまま。
- **artifact-rederived 真値**: `rederived = ci.point_improvement >= margin`。`ci.point_improvement` は replay が stored losses + base_seed から再計算した ci から導出（producer と同一 deterministic bootstrap = loss-derived・`_CI_STAT_BINDINGS` で既に bound）・`margin` は **stamped `material_margin`**（producer が実際に用いた margin = faithful contract）。CLI `--material-margin`（replay ci 構築用 default 0.0）でなく stamped 値を使う = CLI args によらず producer 契約を忠実再現・honest deposit で決して false-positive しない。
- **artifact-when-present-else-skip**: deposit が `is_material` または `material_margin` を stamp しない（いずれかより前の legacy recording）場合は skip（`stale==False`）= 他 gate と同一 posture。legacy で false-positive しない。
- **単一 helper・再利用**: `format_replay`（prose `IS_MATERIAL_STALE` note）と `replay_to_json`（machine field `is_material_stale`）の**両方**が同一 `_is_material_stale(data, ci)` を呼ぶ = §6.2 prose/JSON drift 不能（`3e3aca6`/`6ed0f69`/`37db498`/`371e934`/`47226b9`/`79577a5` と同じ構造）。
- **residual vector の誠実記載**: stored boolean 単独反転（demonstrated reachable corruption）は fail-loud。`point_improvement` を boolean と**同時**反転する coordinated flip は本 gate の scope 外（`point_improvement` 自身は `_ci_stats_stale` で bound なので、その反転は別 gate が検知）= `3e3aca6`（CI bounds も `_ci_stats_stale` で bound する同 posture）と同一の honesty stance。
