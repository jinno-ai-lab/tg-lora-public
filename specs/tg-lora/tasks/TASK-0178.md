## 背景（replay は §4 composite verdict boolean を artifact-rederived 真値に bind しない・TASK-0177 が「transitively verified」と誤認して残置）

`scripts/replay_freeze_validloss_ci.py` は任意の committed deposit を GPU-free で再判定する chokepoint である。producer（`scripts/run_freeze_validloss_ci_9b.result_to_json`）は verdict deposit に §4 composite verdict を stamp する：

```
"verdict": ci.significance_verdict,                  # STRING: SURPASSES / TIES / UNDERSHOOTS
"significant_surpasses": ci.significant_surpasses,    # BOOLEAN (統計軸): significance_verdict == SURPASSES
"is_material": ci.is_material,                        # BOOLEAN (実用軸): point_improvement >= material_margin
"material_margin": ci.material_margin,                # FLOAT (provenance): TASK-0177 stamp
"passes": ci.passes,                                  # BOOLEAN (composite): significant_surpasses AND is_material
```

TASK-0176（`3e3aca6`）は統計軸 boolean を、TASK-0177（`1ed2def`）は実用軸 boolean を、それぞれ artifact-rederived 真値に bind する gate を新設した。TASK-0177 の fix note は明記した：

> `passes`（= `significant_surpasses and is_material`）は両 boolean で transitively に検証され unbound のまま

しかし**両 component boolean gate は各々の component boolean の stored==rederived を検査するだけで、stored `passes` == AND(stored_sig, stored_mat) を検査しない**。これは TASK-0176 が `significant_surpasses` で**既に拒否した transitive-coverage argument の同一再現**である：TASK-0176 も旧 status が「verdict STRING + CI FLOATS が significance を imply するから unbound で可」と主張し、それを：

> stored boolean は独立 field・hand-edit で反転可能だから、他 field が imply しても transitively には覆えない

と **refute して bind した**。`passes` も全く同形（component booleans が imply するが独立 stored field・hand-edit で反転可能）。ゆえに replay の検査は：

- `replay_samples` が losses から verdict を再導出し `faithful` で stored verdict と照合 ✓（**verdict STRING のみ**）
- `_ci_stats_stale` が `_CI_STAT_BINDINGS`（margin-invariant 統計量）を再導出 ✓（**`passes` は margin-dependent として除外**）
- `_significant_surpasses_stale` / `3e3aca6` が stored §7 **統計** boolean を照合 ✓（**component のみ・composite は覆わない**）
- `_is_material_stale` / `1ed2def` が stored §7 **実用** boolean を照合 ✓（**component のみ・composite は覆わない**）

の一方で、**stored `passes` composite boolean を artifact-rederived 真値と照合する検査は一つもなかった**。

ゆえに**内部的に完全に整合した** hand-edited deposit：

- stored `verdict`（STRING）は losses と一致（`faithful:True`）
- stored `significant_surpasses`（統計 boolean）は `ci.significant_surpasses` と一致（`significant_surpasses_stale:False`）
- stored `is_material`（実用 boolean）は `ci.point_improvement >= margin` と一致（`is_material_stale:False`）
- `evidence_hash` は stamp と一致（boolean は evidence key ではない）
- 全 citation / ledger / sub-verdict / Level-1/Level-2 boolean gate が green

が `passes` だけを反転（TIES recording の `passes` を `True` にして composite verdict を**過剰主張**、あるいは SURPASSES を `False` にして**過小主張**）しても、stale field も note も**存在せず完全 silent** に通る。これは corrupt-but-green §4 verdict の**composite verdict claim** を成立させる silent contract-break（GOAL §7）— TASK-0177 が「transitively verified」と誤認して残した named residual。

## 設計（composite-boolean binding via own gate・TASK-0177 transitive-coverage note の refutation・artifact-when-present discipline）

- **TASK-0177 transitive-coverage note の refutation**: 両 component boolean gate は各 boolean の stored==rederived を検査するが stored `passes` == AND(stored_sig, stored_mat) を検査しない = `passes` 単独反転は両 gate が green のまま通る = composite field は component gate では transitively に覆えない = 独立 gate が必要（TASK-0176 が `significant_surpasses` で証明したのと同一論法）。`_significant_surpasses_stale` の旧 docstring「Only passes stays unbound, fully determined by two booleans」も本 refutation に合わせて修正（`passes` は自身の composite gate `_passes_stale` で bind される）。
- **artifact-rederived 真値**: `rederived = ci.significant_surpasses and (ci.point_improvement >= margin)`。`ci.significant_surpasses`（統計軸）は replay ci から直接導出（`3e3aca6` と同一 source）・`ci.point_improvement`（実用軸の入力）は replay が stored losses + base_seed から再計算（`1ed2def` と同一 source）・`margin` は TASK-0177 が producer に stamp させた stamped `material_margin`（producer 契約を忠実再現・CLI args によらない = honest deposit で決して false-positive しない）。
- **producer 側変更不要**: `passes` は既に stamp 済・`material_margin` は TASK-0177 が stamp 済 = replay-gate-only fix（TASK-0177 が producer 変更を要したのと対照的）。
- **artifact-when-present-else-skip**: deposit が `passes` または `material_margin` を stamp しない（いずれかより前の legacy recording）場合は skip（`stale==False`）= 他 gate と同一 posture。legacy で false-positive しない。
- **単一 helper・再利用**: `format_replay`（prose `PASSES_STALE` note）と `replay_to_json`（machine field `passes_stale`）の**両方**が同一 `_passes_stale(data, ci)` を呼ぶ = §6.2 prose/JSON drift 不能（`3e3aca6`/`6ed0f69`/`37db498`/`371e934`/`47226b9`/`79577a5`/`1ed2def` と同じ構造）。
- **residual vector の誠実記載**: stored `passes` 単独反転（demonstrated reachable corruption）は fail-loud。`significant_surpasses` と `point_improvement` を `passes` と**同時**反転する coordinated flip は本 gate の scope 外（両 component は各々の gate で bound なので、その反転は別 gate が検知）= `3e3aca6`/`1ed2def` と同一の honesty stance。`citable_as_full_section4_verdict` は `passes` 反転時に不変（proof of need で実証）= composite claim 偽装は citability に波及しない（誠実記載）。
