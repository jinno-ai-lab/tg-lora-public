## 実装（`scripts/run_freeze_validloss_ci_9b.py`）

- **`_evidence_hash(deposit)` 純関数 + `EVIDENCE_HASH_KEYS`**: raw 測定値（`candidate_losses`/`surrogate_losses`/`control_losses`/`baseline_losses`・`candidate_order`/`control_order`・4 arm の `*_provenance`）+ run-determining config（`model`/`architecture`/`lora_rank_pattern`/`dataset`/`total_steps`/.../`base_seed`）の固定 ordered subset を canonical JSON（`sort_keys`・compact separators）→ SHA-256 hex。**derived label（verdict/gate/regime/CI 統計）は除外**（circular を避ける）。`evidence_hash` 自身を key に含まない → stamping は idempotent。
- **`result_to_json` への stamp**: dict literal → local `deposit = {...}` に変更し、末尾で `deposit["evidence_hash"] = _evidence_hash(deposit)`。新 deposit は全てこの stamp を carry。
- **6 real deposit への backfill**: 既存 committed fixture に `evidence_hash` field を外科的に挿入（evidence byte は 1 byte も変更せず → hash は現状の byte に対して計算）。各 stamp が recompute と一致することを検証済。

## 受け入れ test（`tests/test_run_freeze_validloss_ci_9b.py`・`TestDepositEvidenceHash` +13）

- `test_stamp_matches_recomputed_hash`（6 deposit parametrize）— stamp == recompute（stale/hand-edit 検出）。
- `test_stamp_is_pinned_to_frozen_literal`（6 deposit parametrize）— stamp == 凍結 literal（**coordinated repaint が通せない唯一の test** — repaint が凍結 literal も編集する visible change を強制）。
- `test_hash_is_over_evidence_not_derived_labels` — derived label flip で hash 不変（非 circular 性の mutation 証明）。
- **mutation 証明**: (a) evidence byte edit（stamp 未更新）→ hash 変動 → self-consistency RED。(b) coordinated（stamp も更新）→ 凍結 literal と不一致 → literal RED。(c) derived label flip → hash 不変。

## OPEN（誠実な次の一手・🟡 BLOCKED）

1. **heterogeneous run の run-log 再現** — foreground run を `--log` / tee で file 落ちさせ、loss-curve artifact と共に deposit に参照させる。**現状 content-hash は「committed byte が不変・監査可能」を担保するが、「GPU が byte を生成した」は担保しない**（run log / 独立再現のみが担保）。GPU が空いた窗口で `make freeze-validloss-ci-9b-heterogeneous-generalization` を再実行し run log を capture → 独立 data point として §4 source-of-truth を補強。
2. **future deposit の run-log 同梱** — `run_ci_9b` に run-log path を deposit する仕組み（`run_log_path` field）を追加し、新 deposit が常に再現根拠を carry するようにする（本 iterの content-hash と組み合わせて「独立再現可能」を完全化）。
3. **完全 §4 verdict への残 block（不変）**: (a) heterogeneous leg の full-budget run（`--total-steps 1500 --train-examples 600` generalization regime）・(b) private `src.data` 品質 filter による絶対 loss 比較可能性。
