## 実装詳細

### 1. CUDA メモリガードの追加 🔵

**信頼性**: 🔵 *OOM事象（torch.OutOfMemoryError: Tried to allocate 7.01 GiB）より*

test_task_0027_gpu_env.py の model_and_config フィクスチャにCUDA メモリチェックを追加。`torch.cuda.mem_get_info()` で空きメモリを確認し、不足時は `pytest.skip()` する。

```python
# conftest.py または test_task_0027_gpu_env.py 内
import torch

def _has_enough_vram(required_gb: float = 8.0) -> bool:
    if not torch.cuda.is_available():
        return False
    free, total = torch.cuda.mem_get_info()
    free_gb = free / (1024 ** 3)
    return free_gb >= required_gb
```

**実装ファイル**: `tests/test_task_0027_gpu_env.py`

### 2. model_and_config フィクスチャの堅牢化 🔵

**信頼性**: 🔵 *フィクスチャのOOM事象（11.31 GiB使用中に7.01 GiB割り当て要求）より*

フィクスチャ内でOOMをキャッチし、gracefulにスキップする:

```python
@pytest.fixture(scope="module")
def model_and_config(cfg):
    try:
        model = load_base_model(cfg)
        yield model, cfg
    except torch.OutOfMemoryError:
        pytest.skip("Insufficient GPU memory for model loading")
    finally:
        # クリーンアップ
        torch.cuda.empty_cache()
```

**実装ファイル**: `tests/test_task_0027_gpu_env.py`

### 3. test_vram_under_10gb の修正 🔵

**信頼性**: 🔵 *GPU上の別プロセス（PID 3575810）によるメモリ占有事象より*

VRAMテストに事前チェックを追加し、他プロセスがGPUを使用中の場合はスキップ:

```python
def test_vram_under_10gb(self, model_and_config):
    model, _ = model_and_config
    # 現在のプロセスの割り当て済みメモリのみを検証
    allocated = torch.cuda.memory_allocated() / (1024 ** 3)
    assert allocated < 10.0, f"Model using {allocated:.2f} GiB"
```

**実装ファイル**: `tests/test_task_0027_gpu_env.py`
