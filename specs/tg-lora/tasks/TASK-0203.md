## bug（end-to-end 再現可能）

1. `cycle_state.min_delta`（field doc・cycle_state.py:35-40）は「§5.3 improvement-margin for the ... early-stopping signal」と文書化。`record_full_eval`（同ファイル・TASK-0202）は `if self.best_loss - full_loss > self.min_delta:` で正しく gate し `best_loss`/`stale_cycles` を更新。
2. `record_cycle` は**同じ `best_loss`/`stale_cycles` フィールド**に書くが、旧 `if valid_loss < self.best_loss:` raw 比較（min_delta 無視）。
3. producer は非 full-eval cycle 毎に `cycle_state.record_cycle(..., valid_loss=None if is_full_eval_cycle else loss_pilot)` を呼ぶ（train_tg_lora.py:2659・2831・2960・4056）。`loss_pilot` は `valid_quick_loader` 由来の quick-eval subset loss（2490-2532）。
4. `cycle_state.stale_cycles` は producer で early-stopping 信号として読まれる（4361 `prev_stale` → `_evaluate_full_eval_outcome` → `should_stop`）。`cycle_state.best_loss` は run 公式 `best_valid_loss`/`best_valid_perplexity` として報告（4798/4809/4811）。

結果: **sub-`min_delta` な quick-eval ノイズ改善**（両 production config `early_stopping_min_delta: 0.01` で発火）が non-full-eval cycle で (a) `stale_cycles` を 0 にリセットし early-stopping を無効化（停滞 run が quick-eval ノイズで早期打ち切りを免れ convergence を超過学習 → exit 0）、(b) `best_loss` を下げ報告 `best_valid_loss` を quick-eval subset 値で汚染。`stale_cycles` は §5.3 が管轄する early-stopping 信号なのに `record_cycle` が min_delta gate 無しで書く = TASK-0200 テンプレと同一形状（policy 存在・action site で drop）。

## fix（`src/tg_lora/cycle_state.py`）

`record_cycle` の `if valid_loss < self.best_loss:` を `if self.best_loss - valid_loss > self.min_delta:` へ変更（`record_full_eval` と同一 gate）。`last_valid_loss = valid_loss`（monitoring）は無条件更新のまま温存。戻り値は `None` のまま（caller は戻り値を使わない・save site は `record_full_eval` の `was_new_best` で gate 済み）。

- **byte-identical 性**: `min_delta=0.0`（既定・config 未設定時）で `best_loss - valid_loss > 0.0` ≡ `valid_loss < best_loss` ≡ 従来 raw 比較。fix は `min_delta>0` 設定時（両 production config）のみ発火。strictly-safer（gate を緩めず締めるだけ → spurious new-best を減らすのみ増やさない）。
- **非有限保持**: NaN quick-eval は `best_loss - nan = nan`、+Inf は `best_loss - inf = -inf`、ともに `> min_delta` → False → new-best にならず `stale += 1`（従来 raw `<` と同一・CE loss ≥ 0 なので +inf/NaN が現実的な非有限）。scope は `record_full_eval` の非有限 test と対称。

## 意図的除外（scope hygiene）

- **AST guard 不追加**: TASK-0202 は 5 つの散在 save site を束ねる AST guard を追加したが、`record_cycle` は単一ファイル・単一メソッドなので behavioral test（`TestRecordCycleMinDelta`）が適切な pin（`record_full_eval` も `TestRecordFullEvalMinDelta` behavioral・AST guard 無しと対称）。
- **`-inf` 非有限拡張なし**: `best_loss - (-inf) = +inf > min_delta` → True となり `-inf` を new-best とする潜在 gap は `record_full_eval` と対称（CE loss ≥ 0 で非現実的・TASK-0202 の非有限 test も nan/+Inf のみ）。片側だけ isfinite guard を足すと兄弟と非対称になるため対称維持。
