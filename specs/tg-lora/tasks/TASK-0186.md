## 背景（replay は四軸 gate の budget/regime 格納 label を artifact 再導出値に bind しない・複合 gate が artifact 派生を使うため未照合）

`scripts/replay_freeze_validloss_ci.py` は任意の committed deposit を GPU-free で再判定する chokepoint である。producer の四軸 gate（`freeze_verdict_honesty.full_section4_verdict_gate`）は 4 conjunct を AND し、その結果を `citable_as_full_section4_verdict` として stamp する：

```
citable_as_full_section4_verdict =
    (not proxy_scale)          # scale 軸    — 格納 label bind 済 (_target_scale_label_stale, 37db498)
    and (not reduced_budget)   # budget 軸   — 格納 label 未 bind ← 本 TASK
    and (not is_thin_evidence) # thin 軸     — 格納 label bind 済 (_ci_stats_stale, 6ed0f69)
    and (regime == GENERALIZATION)  # regime 軸 — 格納 label 未 bind ← 本 TASK
```

四軸の内 2 軸（scale / thin）は格納 label を直接 bind する sibling gate が既にある。しかし残り 2 軸（budget / regime）は：

- `_producer_honesty_axes` は `reduced = is_reduced_budget(total_steps, cfg_max_steps)` / `regime = classify_regime(candidate_final_ce_train_loss_mean, ci.candidate_mean)` を**複合 gate の入力として** artifact から再導出するが、**格納 label とは比較しない**。
- `_citation_label_stale`（`d734327`）は複合 boolean `citable_as_full_section4_verdict` のみを再導出値と照合する。格納 `reduced_budget`/`regime` を flip しても複合 gate は artifact 派生を使うため citability は不変 → 複合照合も green のまま。
- `evidence_hash`（`79577a5`）は raw measurements + run config のみを cover し、**derived gate label を意図的に除外**する（`regime`/`reduced_budget` は gate label = 非 evidence）。

ゆえに、格納 per-axis label だけを塗替えた internally-consistent な嘘が全既存 gate green で silent 通過する：

```
# 最悪ケース: citable な full deposit で regime を偽る
regime: 'generalization' -> 'memorization'   # artifacts (ce_train 1.366 vs valid 1.695) は generalization のまま
  faithful=True, citable_as_full_section4_verdict=True（複合 gate は artifact 派生 regime を使用）,
  citation_label_stale=False, ci_stats_stale=[], evidence_hash_stale=False
  => 完全 §4 verdict として引用可能なまま「この run は memorize した」と嘘をつく
```

これは `_target_scale_label_stale`（`37db498`）が scale 軸で閉じたのと**同一 hazard class**（stored axis-label trusted over artifact reality）。本 TASK は budget 軸と regime 軸の 2 つを閉鎖し、**四軸 label family を完成させる**（scale / thin / budget / regime 全ての格納 label が artifact 再導出値に bind）。

## 設計（replay-gate-only・producer 不変・deposit byte-identical）

`scripts/replay_freeze_validloss_ci.py` に torch-free helper 2 つを新設（`_target_scale_label_stale` の sibling・同一 artifact-when-present-else-skip 規律）：

```python
def _reduced_budget_label_stale(data) -> bool:
    if "reduced_budget" not in data: return False
    # total_steps / cfg_max_steps を _producer_honesty_axes と同一の int-coercion で parse
    # （absent/unparsed -> 0 -> reduced = 保守的）。両整数が present のみ bind。
    return bool(data["reduced_budget"]) != is_reduced_budget(total_steps, cfg_max_steps)

def _regime_label_stale(data, ci) -> bool:
    stored, ce = data.get("regime"), data.get("candidate_final_ce_train_loss_mean")
    if stored is None or ce is None: return False
    return stored != classify_regime(ce, ci.candidate_mean)   # leaf = producer の _classify_regime
```

- `format_replay`: `REDUCED_BUDGET_LABEL_STALE` / `REGIME_LABEL_STALE` prose note を `TARGET_SCALE_LABEL_STALE` block の直後に追加（stored / re-derived 値を明示）。
- `replay_to_json`: `reduced_budget_label_stale` / `regime_label_stale` machine field を `target_scale_label_stale` の直後に追加（prose と同一 helper = drift 不可）。
- producer（`run_freeze_validloss_ci_9b.py`）/ fixtures は**一切変更しない** = 全 committed deposit byte-identical・verdict 不変・citable status 不変。
