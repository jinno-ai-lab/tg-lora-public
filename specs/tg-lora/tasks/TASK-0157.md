## 背景（前 iter までの verdict が測れなかった軸）

前 iter まで（`49d36d8` → `32f128d` → `e8abf47` → `4749840`）で homogeneous leg の real 9B §4 A/B verdict は **3 本立ち**（surrogate / direction / generalization・全 non-thin SURPASSES・direction-isolated・regime-robust）に確定した。しかし**3 つとも freeze-vs-freeze** A/B だった: candidate（output-first 凍結）vs surrogate（random 凍結）vs control（input-first 凍結）。ゆえに「output-first 凍結が*他の凍結順*に勝つ」ことは示せるが、GOAL §4 line 247 の condition (a)「凍結が **FULL backprop** に対して valid_loss を犠牲にするか」は**構造的に測れない**まま残っていた — surrogate も凍結するため baseline ではなく、freeze 全体の絶対品質コストへの問いに答えられない。

**本 iter の問題意識**: GOAL §4 の success criterion は condition (a) と condition (b) の**両立**を要求する。sibling 3 deposit は (b) のみを閉じ、(a) は未測定だった。完全 §4 verdict は依然 private `src.data` + full `max_steps=1500` で物理 block されるが、condition (a) の**方向**（凍結が full backprop に劣るか・同等か・優るか）は、no-freeze full-CE baseline arm（GOAL §4 line 247 control (i)）を加えるだけで GPU 上で直接測定できる。これは feedback の「SINGLE highest-leverage non-edge-case task」に合致 — verdict instrument の polish ではなく、§4 verdict の**残りの中核軸**そのものを取得する。

## 実装

### 1. no-freeze baseline arm（`scripts/run_freeze_validloss_ci_9b.py` `run_ci_9b`）

`arm_valid_loss_9b(..., depth=0)` で純粋な full-backprop control を構成: `FreezeScheduleConfig(max_depth=0)` → 0 凍結計画 → `ctrl.frozen_layers` は常に空 → `use_local_loss and ctrl.frozen_layers` が常に偽 → **常に full-CE branch**（`out.loss`）= active scope 全層を task loss で最後まで訓練。`depth=0` は `freeze_schedule.py` で合法（`max_depth<0` のみ reject・line 203-204）。seed offset `base_seed+300+i`（candidate `+i` / surrogate `+100+i` / control `+200+i` と band 衝突なし・`test_baseline_ci_uses_distinct_seed_offset_from_all_other_arms` で pin）。OOM 対策（load-once / in-place LoRA reset per arm）は baseline arm にも適用 — baseline は max-compute arm（full scope を毎 step 訓練）だが in-place reset で 2 機目の model load を回避し seq1024 で完走。

### 2. baseline CI + honesty relabel

`baseline_ci = surrogate_valid_loss_ci(candidate_losses, baseline_losses)` — generic pairwise bootstrap の "surrogate" slot に baseline を流す。`_baseline_ci_to_json` は baseline の `surrogate_mean` を `baseline_mean`・`n_surrogate` を `n_baseline` に relabel（no-freeze control を random surrogate と読み間違えないよう・direction arm の `control_mean` relabel と対称）。verdict reading（§4 line 247）: SURPASSES（candidate < baseline・凍結が full backprop を上回る）/ TIES（candidate ≈ baseline・tolerance 内 = §4 target）/ UNDERSHOOTS（candidate > baseline・凍結が quality を犠牲に）。`--n-baseline` flag（default 0 = byte-identical to before）・Makefile `make freeze-validloss-ci-9b-baseline`（`FREEZE_9B_BASELINE_FLAGS`・新規 deposit `tests/fixtures/freeze_validloss_ci_9b_baseline.json`）。

### 3. 受け入れ test（`tests/test_run_freeze_validloss_ci_9b.py`・+11・mutation 証明）

`TestBaselineControlDeposit`（+11）— (1) fixture real target-scale / (2) non-thin baseline block / (3) **baseline arm never freezes**（`frozen_layers==[]`・`n_trainable_params==scope_trainable_params`・control 完全性の load-bearing pin）/ (4) baseline が full-CE 全貫（`last_train_loss>0.5`・`final_ce/last` 比 0.1-10・frozen arm とは >1000x gap で識別）/ (5) surrogate verdict replay / (6) **baseline verdict replay**（記録 float が deterministic bootstrap で記録 verdict を再獲得・verdict-agnostic）/ (7) candidate pool 共有 / (8) **candidate beats full backprop** headline pin / (9) **baseline overfits / freeze regularizes** mechanism pin / (10) generalization regime（全 4 arm `final_ce>0.5`）/ (11) non-thin 全 arm。

### 完全 4-arm ランキング（lower valid_loss = better）

| arm | order | frozen layers | valid_loss mean | final_ce_train_loss | train-valid gap |
|---|---|---|---|---|---|
| **candidate** | output-first | {29,30,31} | **1.5148** | 1.5074 | 0.007 |
| surrogate | random | {25,28,31}/{26,28,29}/{24,27,28} | 1.5270 | 1.5390 | 0.012 |
| control | input-first | {24,25,26} | 1.5376 | 1.5592 | 0.022 |
| **baseline** | **no freeze (full backprop)** | **{}** | **1.5401** | **0.7741** | **0.766** |

- **candidate-vs-full-backprop-baseline A/B**（§4 condition (a)）: **SURPASSES**・point **0.0253**・CI[95%] **[0.018, 0.029]**・non-thin（3/arm）・CI は zero を完全に除外。凍結が quality を犠牲にせず** full backprop を上回る**（condition (a) は tolerance 内どころか SURPASSES で充足）。
- **完全 4-arm 単調順位**: candidate 1.5148 < random surrogate 1.5270 < input-contig control 1.5376 < full backprop 1.5401。surrogate / direction A/B も同一 session で再取得し SURPASSES を再現（前 iter deposit と一致・独立確認）。
- **mechanism（freeze-as-regularizer）**: full-backprop baseline は train set への fit が**強い**（`final_ce_train_loss` ≈ **0.7741**）のに held-out valid は**最悪**（1.5401）= **過学習**（train-valid gap 0.766）。対して凍結 3 arm は train CE ≈ valid CE ≈ 1.5（gap ≤0.022 = 真の汎化）。凍結＋activation-matching local loss が adapter の train-fit 強さを CE ~1.5 に cap し、それが**正則化**として効いて full backprop より汎化した — 「candidate が baseline に勝つ」は noise でなく過学習抑制のシグナチャ。baseline provenance: `frozen_layers=[]`・`n_trainable_params=10819584`（= scope 全量・`depth=0` で凍結ゼロの直接証明）・`last_train_loss` ~2.18（full-CE magnitude・frozen arm の ~1e-4 local loss と対照的）。
