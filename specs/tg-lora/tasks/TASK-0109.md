## 主要実装項目

### G4評価ロジック拡張 🔵

**信頼性**: 🔵 *paper_experiment_plan.md G4定義・evaluate_paper_gates.py _check_g4()現在実装より*

evaluate_paper_gates.pyの_check_g4()をinformationalから実際の判定ロジックに拡張:
- 入力: cold summary JSON, warm summary JSON, train-cache on summary JSON, train-cache off summary JSON
- G4.1: warm speedupが全seedで正
- G4.2: cold build vs warm load差が明確（cold >> warm）
- G4.3: train-cache on の memory/frontier effect が off より強い

**実装ファイル**: `scripts/evaluate_paper_gates.py`, `tests/test_evaluate_paper_gates.py`

### Ablation入力インターフェース 🔵

**信頼性**: 🔵 *既存run_ablation_suite.sh・compare_paper_memory_modes.pyパターンより*

- 複数summary JSONを入力として受け取るCLI引数追加
- --cold-summary, --warm-summary, --cache-on-summary, --cache-off-summary

## 基本的なテスト要件

### G4 Gate判定テスト 🔵

**信頼性**: 🔵 *evaluate_paper_gates.py既存テストパターンより*

- G4.1 pass: warm speedup全seed正
- G4.1 fail: 1seed以上warm speedup <= 0
- G4.2 pass: cold >> warm差が明確
- G4.3 pass: cache-on > cache-off
- 混在パターンの正確な判定

**テストファイル**: `tests/test_evaluate_paper_gates.py`
