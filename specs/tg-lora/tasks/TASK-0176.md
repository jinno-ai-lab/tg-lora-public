## 背景（replay は §7 significance boolean を ci.significant_surpasses に bind しない）

`scripts/replay_freeze_validloss_ci.py` は任意の committed deposit を GPU-free で再判定する chokepoint である。producer（`scripts/run_freeze_validloss_ci_9b.result_to_json`）は verdict deposit に §7 の両軸を stamp する：

```
"verdict": ci.significance_verdict,                # STRING: SURPASSES / TIES / UNDERSHOOTS
"significant_surpasses": ci.significant_surpasses, # BOOLEAN: significance_verdict == SURPASSES
# is_material / passes は producer-stamped だが margin-dependent なので bind 対象外
```

`significant_surpasses` は §7 が**統計的有意性と実用的集中度を区別**する前者の first-class claim であり、reader / table / downstream consumer が「この lead は統計的に有意か？」として cite する boolean（CI が zero を排除する = `significance_verdict == SURPASSES`）。しかし replay の検査は：

- `replay_samples` が losses から verdict を再導出し `faithful` で stored verdict と照合 ✓（**verdict STRING のみ** — boolean は覆わない）
- `_ci_stats_stale` / `6ed0f69` が `_CI_STAT_BINDINGS`（lower/upper/means/point/confidence/n/thin = margin-invariant 統計量）を ci から再導出して照合 ✓（**CI FLOATS のみ** — `significant_surpasses` は binding list に入っていない）
- `evidence_hash_stale` が raw evidence bytes の SHA-256 を再導出 ✓（**raw evidence のみ — gate label は意図的に除外**・`freeze_evidence_hash.py:37` "NEVER the derived verdict/gate/regime labels"）

の一方で、**stored `significant_surpasses` boolean を `ci.significant_surpasses` から再導出して照合する検査は一つもなかった**。`significant_surpasses` は verdict label（`faithful` が覆う）や CI floats（`6ed0f69` が覆う）が imply する派生量だが、**どちらの binding list にも属さない独立の stored field** なので、両 gate が green でも boolean の hand-edit を transitively に覆わない。

ゆえに**内部的に完全に整合した** hand-edited deposit：

- stored `verdict`（STRING）は losses と一致（`faithful:True` — label は触らない）
- stored `lower`/`upper`/means（FLOATS）は ci と一致（`ci_stats_stale:[]` — floats は触らない）
- `evidence_hash` は stamp と一致（boolean は evidence key ではない・gate label だから除外）
- 全 citation / ledger / sub-verdict / Level-1 boolean gate が green

が `significant_surpasses` だけを反転（TIES/UNDERSHOOTS recording を `True` にして有意性を**過剰主張**、あるいは SURPASSES を `False` にして**過小主張**）しても、stale field も note も**存在せず完全 silent** に通る。これは corrupt-but-green §4 verdict の**§7 統計有意性 claim** を成立させる silent contract-break（GOAL §7）。

**到達性（実証済 proof of need・`test_corrupt_flag_passes_every_other_gate`）**: committed `freeze_validloss_ci_9b_full.json`（recorded **TIES**・stored `significant_surpasses=False`）の boolean を `True` に反転すると、`faithful=True`・`ci_stats_stale=[]`・`evidence_hash_stale=False`・`citation_label_stale=False`・`ledger_losses_stale=[]`・`direction_verdict_stale=False`・`target_scale_label_stale=False` の**全既存 gate green のまま**、`significant_surpasses_stale` のみが `True` を捕まえる（TIES を SURPASSES に偽装する危険方向）。

## 設計（ci.significant_surpasses への binding・artifact-when-present discipline・margin-axis 区別）

- **同 class・異 field**: `37db498`（Level-1 citation boolean）と同 class（stored boolean を artifact-derived 真値に bind）だが照合対象は異なる（`not proxy_scale`〔1-term deposit field〕でなく `ci.significant_surpasses`〔ci から再導出〕）。`6ed0f69`（derived 統計量）とは**直接 sibling**（同じ `ci` から再導出・同じ `_ci_stats_stale` の直後に配置）だが FLOATS でなく BOOLEAN を照合する。`79577a5`/`47226b9`/`371e934` とは異なる照合対象。
- **margin-invariant だから bind 可能**: `significant_surpasses` = `significance_verdict == SURPASSES` は **margin-invariant**（CI の zero 排除のみで決まり `material_margin` に依存しない）なので deterministic に再導出可能。対照的に `is_material`（`point_improvement >= material_margin`）と `passes`（`significant_surpasses and is_material`）は **margin-dependent** で `material_margin` が deposit に stamp されないので bind 対象外 — `_CI_STAT_BINDINGS` が `is_material` を除外したのと同理由。ゆえに `significant_surpasses` のみを bind し `is_material`/`passes` は unbound のまま（両者は producer-stamped 参照用・verdict STRING + CI FLOATS + significance BOOLEAN の 3 連で既に transitively に検証される）。
- **artifact-when-present-else-skip**: deposit が `significant_surpasses` を stamp しない（field より前の legacy recording）場合は skip（`stale==False`）= 他 gate と同一 posture。legacy で false-positive しない。
- **単一 helper・再利用**: `format_replay`（prose `SIGNIFICANT_SURPASSES_STALE` note）と `replay_to_json`（machine field `significant_surpasses_stale`）の**両方**が同一 `_significant_surpasses_stale(data, ci)` を呼ぶ = §6.2 prose/JSON drift 不能（`6ed0f69`/`37db498`/`371e934`/`47226b9`/`79577a5` と同じ構造）。

## fix

`scripts/replay_freeze_validloss_ci.py`:

1. `_significant_surpasses_stale(data, ci)` — `stored = data.get("significant_surpasses")`・`stored is None` なら `False`（skip）。`bool(stored) != ci.significant_surpasses` なら `True`。`ci.significant_surpasses` は replay が stored losses + base_seed から再計算した ci から導出（producer と同一 deterministic bootstrap）。
2. `format_replay`: `CI_STATS_STALE` note の直後に `SIGNIFICANT_SURPASSES_STALE` note（stale 時のみ・対処＝fresh producer run で re-stamp / stored boolean 修正）を出力。note は stored 値・artifact-rederived 値・読替結果（`significant (SURPASSES)` / `NOT significant`）を明示。
3. `replay_to_json`: return dict で `ci_stats_stale` の直後に `significant_surpasses_stale` field を追加。

保証: **stored §7 significance boolean が `ci.significant_surpasses` と乖離する internally-consistent deposit は `significant_surpasses_stale=True` + `SIGNIFICANT_SURPASSES_STALE` note で fail-loud**。committed corpus 全体（SURPASSES-stored-True + TIES-stored-False + UNDERSHOOTS-stored-False の 3 verdict class）で stored boolean == `ci.significant_surpasses`（byte-identical）なので `significant_surpasses_stale==False`。gate と prose は同一 helper 経由で drift しない（§6.2）。
