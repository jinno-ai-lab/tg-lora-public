## タスク概要

M10.3 disk-death guard ファミリ（db3c08f → 3c2c9d7 → d2218ed → 43acef0 → c96a5a0）は、`checkpoint-cycle-*` ディレクトリの無限蓄積ベクトルを閉じ、knob をスキーマ gate で固定し、周期保存経路に seam を抽出し、発射性を repo-wide gate 化した。しかし本ランには**第2の無限蓄積ベクトル**が存在し、見逃されていた。

`train_tg_lora` は `cfg.training.save_trajectory_delta_artifacts: true`（M10 両 config が出荷、L40）のとき、`save_trajectory_delta_artifact` でサイクル毎に1〜2個の `.pt` ファイルを `run_dir/trajectory_delta_artifacts/` へ書く（pilot 成功時 `after_pilot`・speculative update 時 `after_speculative_update`、train_tg_lora.py L2104/L3209）。`trajectory_delta_artifact_interval` は既定 1（L1293）。これらを**削除する経路はコード上どこにもない**。

cycle 保護 `prune_checkpoint_cycles` の regex は `^checkpoint-cycle-(\d+)$`（`_CYCLE_DIR_RE`）であり、disk-floor (`min_free_disk_gb`) も `checkpoint-cycle-*` のみ走査する。よって artifacts は cycle 保護の傘外である。M10 ラン（`save_every_cycles: 1`・`max_cycles: 120`・`save_trajectory_delta_artifacts: true`）では、最大約 240 個の delta-tensor ファイルがサイクルに比例して線形蓄積し、**`keep_last_checkpoints` が既に `3` でオンであるにもかかわらず**回収されない。これは「保護が存在するが、このベクトルには発火しない」クラスそのものであり、TASK-0131〜0136 が閉じたかった無限蓄積クラスの、同一ラン上の未閉包残渣である。

本タスクはこの残渣を閉じる。新規 knob は増やさない — 既存の `keep_last_checkpoints` / `min_free_disk_gb` を読む `prune_trajectory_delta_artifacts(_from_cfg)` を追加し、cycle 保護と**同一 knob で両ベクトルを抑える**。M10 config は既に knob をオンにしているため、**config 変更なしで**第2ベクトルも抑えられる。cycle 保護と artifact 保護は、共通の `_bound_keyed_paths` core（count-bound + disk-floor over `(key, Path)`）で同じ方針を共有する。
