## タスク概要

TASK-0144 は gate 評価器に「未測定 gate を反証（disproven）と区別する INSUFFICIENT EVIDENCE 第3状態」を導入し、TASK-0145 はそれを下流 claim-level 報告へ伝播した。両者とも in-process unit test（合計 8 + 13 件）で非不活性を証明した。しかし **その honesty contract を「ユーザが CLI を叩いたとき本当にそう振る舞うか」で検証する経路は完全に死んでいた**: `evaluate_paper_gates.py` は `from src.analysis.stats import ...`（L32）するが repo-root bootstrap を持たず、bare の `python scripts/evaluate_paper_gates.py …` 起動では Python が `scripts/`（cwd ではない）を sys.path に置くため `src.*` が解決せず、即座に `ModuleNotFoundError: No module named 'src.analysis'` で落ちる。

これは TASK-0140〜0145 が閉じてきた「コードは直ったように見えるが実は不活性」という静かな状態の**検証側変種**である: 機能は unit test で正しく実装されているが、それを end-to-end で pin する CLI test 群（TASK-0144 自身が書いた `TestInsufficientEvidenceHonesty` の E2E 3件を含む 13 件）は全て import 段で死んでおり、退出 code の誠実化（default exit 0 / `--strict` exit 1）・JSON の `insufficient_evidence_gates`/`disproven_fail_gates` 分離・第3状態の末端伝播を**実 CLI で一度も検証できていなかった**。さらに Makefile の `paper-memory-evaluate-gates` target（L435）も PYTHONPATH prefix なしで本 script を起動するため、documented な user-facing entry point 自体が壊れていた。

解決は sibling script 群が既に確立している idiom 1 行: 起動時に `sys.path.insert(0, str(Path(__file__).resolve().parents[1]))` で repo-root を importable にする。新規依存・新規 knob なし。これで (a) bare CLI が発射する、(b) Makefile target が発射する、(c) 13 件の CLI/honesty test が初めて実際の CLI 振る舞いを検証して緑になる。
