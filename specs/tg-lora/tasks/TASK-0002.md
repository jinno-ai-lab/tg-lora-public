## 主要実装項目

### カバレッジギャップ分析 🔵

**信頼性**: 🔵 *既存テスト構造から直接確認可能*

`pytest --cov=tg_lora --cov-report=term-missing` を実行し、各モジュールの未カバー行を特定する。特に以下を確認:

- `extrapolator.py`: `cap_update` の NaN/Inf 分岐
- `velocity.py`: `cosine_similarity` のエッジケース
- `random_walk_controller.py`: `propose()` の全ランダム分岐
- `lora_utils.py`: `set_trainable_lora_layers`・`configure_trainable_lora_scope`

**実装ファイル**: `tests/`

### エッジケーステストの追加 🔵

**信頼性**: 🔵 *既存テストパターンに基づく*

- ゼロテンソルに対する velocity 更新
- 空の LoRA パラメータセット
- 極端な alpha/beta 値（alpha=0, beta=1.0 など）
- 単一層モデルでの層選択
- 非有限値を含む velocity での外挿

**実装ファイル**: `tests/test_edge_cases.py`（新規）

### パラメータ化テストの導入 🟡

**信頼性**: 🟡 *テストパターンの推測を含む*

主要な入力バリエーションを `@pytest.mark.parametrize` で整理し、テストの重複を減らしつつ網羅性を向上させる。

**実装ファイル**: `tests/`（既存ファイルの更新）
