## 実装詳細

### 1. TrainingAdvisorクラス 🔵

**信頼性**: 🔵 *CycleMonitor・TrajectoryAnalyzer既存API統合*

`src/tg_lora/training_advisor.py`に新規モジュールを作成。CycleMonitorの発散・停滞検知とTrajectoryAnalyzerの収束・異常検知を統合し、優先度付きアクションリストを生成する。

**実装ファイル**: `src/tg_lora/training_advisor.py`

**主要メソッド**:
- `__init__(config: AdvisorConfig)`: CycleMonitor + TrajectoryAnalyzer初期化
- `evaluate(cycle, train_loss, valid_loss, ...) -> AdvisoryReport`: 統合評価
- `summary() -> dict`: アドバイザ状態出力

**アクション生成ルール**:
- NaN/Inf検出 → rollback + reduce_lr（critical）
- Loss spike → reduce_lr（high）
- 停滞 → increase_k（high）
- 異常 → reduce_lr + adjust_alpha（medium）
- 下降トレンド → increase_lr（low）

### 2. AdvisoryActionデータクラス 🔵

**信頼性**: 🔵 *設計意图から直接導出*

action_type（reduce_lr/increase_lr/stop_training/save_checkpoint/increase_k/decrease_k/adjust_alpha/rollback/resume/no_action）、priority（critical/high/medium/low）、reason、suggested_value、confidence（0.0~1.0）。

### 3. AdvisoryReportデータクラス 🔵

**信頼性**: 🔵 *CycleMonitor HealthReport + TrajectoryAnalyzer TrajectoryReportの統合*

overall_health（healthy/warning/critical）、actions（優先順位ソート済み）、summary、cycle_health、trajectory_summary、timestamp。top_action()メソッドで最高優先度アクションを返却。

### 4. advise_training.py CLI 🔵

**信頼性**: 🔵 *analyze_trajectory.pyのCLIパターンに準拠*

`scripts/advise_training.py`にCLIツールを作成。run_metrics.jsonlからcycle recordsを抽出し、generate_advice_from_history()でAdvisoryReportを生成。

**実装ファイル**: `scripts/advise_training.py`

**CLI引数**:
- 位置引数: run_metrics.jsonlパス
- `--json`: JSON出力フラグ
- `-o`: 出力ファイルパス
- `--patience`: 停滞patience値
- `--spike-threshold`: スパイク閾値
- `--trajectory-window`: 軌跡分析ウィンドウ幅

**Exit codes**: 0=正常, 1=ファイルなし/レコードなし, 2=クリティカル状態
