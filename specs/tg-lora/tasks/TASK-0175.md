## 背景（replay は Level-1 citation boolean を not proxy_scale に bind しない）

`scripts/replay_freeze_validloss_ci.py` は任意の committed deposit を GPU-free で再判定する chokepoint である。producer（`scripts/run_freeze_validloss_ci_9b.result_to_json` line 2083）は **Level-1 citation boolean** を deposit 自身の `proxy_scale` field の deterministic 関数として stamp する：

```
"citable_as_target_scale": (not result["proxy_scale"]),
```

これは reader が最初に確認する **Level-1 citation boolean**（「これは真の target-scale 9B recording か？」）であり、Level-2 `citable_as_full_section4_verdict` gate が上に compose する前提である。しかし replay の検査は：

- `replay_samples` が losses から verdict を再導出し `faithful` で stored verdict と照合 ✓（**verdict label のみ**）
- `citation_label_stale` が `citable_as_full_section4_verdict`（**Level-2 boolean**）を 4-axis artifact から再導出して照合 ✓（**Level-2 のみ — Level-1 boolean は独立 field**）
- `evidence_hash_stale` が raw evidence bytes の SHA-256 を再導出 ✓（**raw evidence のみ — gate label は意図的に除外**・`freeze_evidence_hash.py:37` "NEVER the derived verdict/gate/regime labels"）

の一方で、**stored Level-1 boolean `citable_as_target_scale` を `not proxy_scale` から再導出して照合する検査は一つもなかった**（`replay_to_json` は自身の出力用に 3-term の `citable_as_target_scale` を再導出するが stored 値とは比較しない）。

ゆえに**内部的に完全に整合した** hand-edited deposit：

- stored `verdict` は losses と一致（`faithful:True` — losses は触らない）
- stored `citable_as_full_section4_verdict`（Level-2）は artifact と一致（`citation_label_stale:False` — Level-2 boolean は触らない）
- `evidence_hash` は stamp と一致（Level-1 boolean は evidence key ではない・gate label だから除外）
- 全 label / sub-verdict / ledger / stats gate が green

が `citable_as_target_scale` だけを反転（proxy recording を `True` にして target-scale を**過剰主張**、あるいは 9B run を `False` にして**過小主張**）しても、stale field も note も**存在せず完全 silent** に通る。これは corrupt-but-green §4 verdict の**Level-1 citation claim** を成立させる silent contract-break（GOAL §7）。Level-2 gate は Level-1 を覆わない（Level-2 boolean は reduced-context / thin / non-generalization で honestly `False` になり得るので、Level-2 が `False` でも Level-1 の hand-edit は独立に通る）。

**到達性（実証済 proof of need・`test_corrupt_label_passes_every_other_gate`）**: committed `freeze_validloss_ci_9b_full.json`（recorded **TIES**・`citable_as_target_scale=True`・`proxy_scale=False`）の `citable_as_target_scale` を `False` に反転すると、`faithful=True`・`evidence_hash_stale=False`・`citation_label_stale=False`・`ledger_losses_stale=[]`・`direction_verdict_stale=False` の**全既存 gate green のまま**、`target_scale_label_stale` のみが `True` を捕まえる。

## 設計（producer の 1-term 契約への binding・artifact-when-present discipline）

- **新 class**: `d734327`（Level-2 citability boolean）・`6ed0f69`（derived 統計量）・`79577a5`（raw evidence hash）・`47226b9`（外部 ledger losses）・`371e934`（verdict label）は全て異なる照合対象。本 TASK は「**deposit の stored Level-1 citation boolean vs 同 deposit の `proxy_scale` から再導出した `not proxy_scale`**」= 同 chokepoint（replay gate）・同 §6.2 prose/JSON drift 不能構造だが**異 class**（照合対象が Level-1 citation boolean）・`d734327` の**直接 Level-1 sibling**。
- **producer の exact 1-term 契約**: binding は `stored == (not data["proxy_scale"])` のみ。producer は `not proxy_scale`（1-term）を stamp する。replay は自身の出力用に `not proxy_scale and not synthetic and not negative_control`（3-term・stricter）を計算するが、これは**異なる gate**（synthetic / negative-control recording を追加で除外）であり stored field の契約ではない。stored field を 3-term に bind すると synthetic / neg-control の target-scale deposit で false-positive する（producer は 1-term を stamp するので）。ゆえに producer の exact 1-term 不変量に bind する = honest producer-stamped deposit で決して false-positive しない（全 committed fixture で byte-identical・機械証明済）。
- **artifact-when-present-else-skip**: deposit が `citable_as_target_scale` を stamp しない（field より前の legacy recording）場合は skip（`stale==False`）= 他 gate と同一 posture。legacy で false-positive しない。
- **単一 helper・再利用**: `format_replay`（prose `TARGET_SCALE_LABEL_STALE` note）と `replay_to_json`（machine field `target_scale_label_stale`）の**両方**が同一 `_target_scale_label_stale(data)` を呼ぶ = §6.2 prose/JSON drift 不能（`d734327`/`371e934`/`47226b9`/`79577a5`/`6ed0f69` と同じ構造）。

## fix

`scripts/replay_freeze_validloss_ci.py`:

1. `_target_scale_label_stale(data)` — `stored = data.get("citable_as_target_scale")`・`stored is None` なら `False`（skip）。`bool(stored) != (not bool(data.get("proxy_scale", True)))` なら `True`。
2. `format_replay`: `CITATION_LABEL_STALE` note の直後に `TARGET_SCALE_LABEL_STALE` note（stale 時のみ・対処＝fresh producer run で re-stamp / stored boolean 修正）を出力。
3. `replay_to_json`: return dict で `citation_label_stale` の直後に `target_scale_label_stale` field を追加。

保証: **stored Level-1 boolean が `not proxy_scale` と乖離する internally-consistent deposit は `target_scale_label_stale=True` + `TARGET_SCALE_LABEL_STALE` note で fail-loud**。committed corpus 全体（9B stored-True + proxy stored-False 両方向）で stored boolean == `not proxy_scale`（byte-identical）なので `target_scale_label_stale==False`。gate と prose は同一 helper 経由で drift しない（§6.2）。
