### (2) commit-hygiene guard（item #2）— `scripts/check_substantive_diff.py`

git 自身の whitespace-only 定義を忠実に再現する純粋関数 + CLI wrapper：
- `content_line_counts(diff)` — unified diff の added/removed *content* 行数を計数（`+++`/`---`/`@@`/`diff `/`index `/mode/rename/`\` 等の構造行を除外）。
- `substantive_verdict(raw, ws_ignored)` — 純粋関数。raw が空 → `empty`〔exit 3〕・raw に content あるが `git diff -w --ignore-blank-lines` 相当が空 → `normalization-only`〔exit 4〕・それ以外 → `substantive`〔exit 0〕。
- `_run_git_diff` + `main()` — デフォルト `HEAD^..HEAD`（最終 commit = judge 対象）、`--range A..B`・`--staged`（index = pre-commit）対応。非ゼロ exit で skip(3)/squash(4)/accept(0) を呼出側が弁別可能。
- `make check-commit-substantive`〔`RANGE=`/`STAGED=1` override 可〕で呼出可能。`ci:` には強制組込せず（無関係 CI run を壊さない）・make-run loop が judge 前に invoke する tool として提供。
