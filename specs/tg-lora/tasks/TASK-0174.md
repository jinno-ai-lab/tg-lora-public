## 背景（replay は cited される derived 統計量を再導出しない）

`scripts/replay_freeze_validloss_ci.py` は任意の committed deposit を GPU-free で再判定する chokepoint である。producer（`scripts/run_freeze_validloss_ci_9b.result_to_json` lines ~1995-2000）は verdict を計算した `ci` オブジェクトから derived 統計量を**直書き**する：

```
candidate_mean / surrogate_mean / point_improvement / lower / upper /
confidence / is_thin_evidence / n_candidate / n_surrogate / is_material
```

これらは §4 verdict deposit の**定量的裏付け**であり、論文の結果表に「candidate 1.71804 vs surrogate 1.71903, CI[95%]=[-0.001,+0.003], improvement=0.001」のように cited される。しかし replay の検査は：

- `replay_samples` が losses から verdict を再導出し `faithful` で stored verdict と照合 ✓（**verdict label のみ**）
- `evidence_hash_stale` が raw evidence bytes の SHA-256 を再導出 ✓（**raw evidence のみ — derived 統計量は意図的に除外**）
- `citation_label_stale` / `*_verdict_stale` / ledger binding ✓

の一方で、**stored derived 統計量を replay の `ci` から再導出して照合する検査は一つもなかった**（`replay_to_json` は `ci.candidate_mean` 等を re-derive して出力するが stored 値と比較しない）。

ゆえに**内部的に完全に整合した** hand-edited deposit：

- stored `verdict` は losses と一致（`faithful:True` — losses は触らない）
- `evidence_hash` は stamp と一致（derived 統計量は evidence key ではない）
- 全 label / sub-verdict / ledger gate が green

が `point_improvement=0.05` / `lower=0.02` / `upper=0.08` / `candidate_mean=1.50`（confidently-positive な結果）を主張しても、stale field も note も**存在せず完全 silent** に通る。これは corrupt-but-green §4 verdict の**定量的主張**を成立させる silent contract-break（GOAL §7）。

**到達性（実証済 proof of need・`test_corrupt_stats_pass_every_other_gate`）**: committed `freeze_validloss_ci_9b_full.json`（recorded **TIES**・`citable_as_full_section4_verdict=True`）の `point_improvement`/`lower`/`upper` を上書きすると、`faithful=True`・`evidence_hash_stale=False`・`citation_label_stale=False`・`ledger_losses_stale=[]` の**全既存 gate green のまま**、`ci_stats_stale` のみが `['candidate_mean','point_improvement','lower','upper']` を捕まえる。

## 設計（margin-invariant 統計量の binding・artifact-when-present discipline）

- **新 class**: `79577a5`（raw evidence hash）・`47226b9`（外部 ledger losses）・`d734327`（citable boolean）・`371e934`（verdict label）は全て異なる照合対象。本 TASK は「**deposit の stored derived 統計量 vs 同 deposit の losses から再導出した `ci`**」= 同 chokepoint（replay gate）・同 §6.2 prose/JSON drift 不能構造だが**異 class**（照合対象が derived 定量統計量）。
- **margin-invariant 範囲**: `_CI_STAT_BINDINGS` は `is_material` を**意図的に除外**する。`is_material` は `point_improvement >= material_margin`（`freeze_surrogate_ci.py:173`）で唯一 `material_margin` に依存し、margin は deposit に stamp されないため、非 zero margin の producer run で false-positive する。残り 9 key（means / CI bounds / point improvement / confidence / sample sizes / thin-evidence flag）は全て margin-invariant であり、全 committed deposit で byte-identical に bind する。
- **artifact-when-present-else-skip**: deposit が当該 key を stamp しない（proxy/legacy）場合は skip（`stale==[]`）= 他 gate と同一 posture。proxy/legacy で false-positive しない。
- **単一 helper・再利用**: `format_replay`（prose `CI_STATS_STALE` note）と `replay_to_json`（machine field `ci_stats_stale`）の**両方**が同一 `_ci_stats_stale(data, ci)` を呼ぶ = §6.2 prose/JSON drift 不能（`371e934`/`d734327`/`47226b9`/`79577a5` と同じ構造）。

## fix

`scripts/replay_freeze_validloss_ci.py`:

1. `_CI_STAT_BINDINGS` 定数 — `(stored_key, ci_attr)` の 9 対（`is_material` 除く）。
2. `_ci_stats_stale(data, ci)` — 各 key について `data[stored_key] != getattr(ci, ci_attr)` なら stale list に追加。key 不在時は skip。空 list は「全一致 or stamp なし」。
3. `format_replay`: sub-verdict staleness loop の直後に `CI_STATS_STALE` note（stale 時のみ・対処＝fresh producer run で re-stamp）を出力。
4. `replay_to_json`: return dict で `evidence_hash_stale` の直後に `ci_stats_stale` field を追加。

保証: **cited derived 統計量が losses と乖離する internally-consistent deposit は `ci_stats_stale` に統計量名 + `CI_STATS_STALE` note で fail-loud**。committed corpus 全体で stored 統計量 == re-derived（byte-identical）なので `ci_stats_stale==[]`。gate と prose は同一 helper 経由で drift しない（§6.2）。
