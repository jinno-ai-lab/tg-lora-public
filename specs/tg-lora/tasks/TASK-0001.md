## タスク概要

GitHub Actions による CI/CD パイプラインを構築し、プルリクエスト・マージ時に自動的に lint・テスト・カバレッジ計測を実行する。これにより、コード品質の継続的な担保とリグレッションの早期検出を実現する。

## 主要実装項目

### CI ワークフロー定義 🟡

**信頼性**: 🟡 *標準的な GitHub Actions パターン*

`.github/workflows/ci.yml` に以下のジョブを定義する:

1. **lint ジョブ**: `ruff check .` と `ruff format --check .`
2. **test ジョブ**: `pytest --cov=tg_lora --cov-report=xml` を Python 3.11/3.12 の matrix で実行
3. **依存関係キャッシュ**: `pip` キャッシュで実行時間を短縮

**実装ファイル**: `.github/workflows/ci.yml`

### Makefile のテストタスク追加 🔵

**信頼性**: 🔵 *既存 Makefile パターンに基づく*

既存 Makefile に `test`・`lint`・`format` ターゲットを追加し、ローカルでも CI と同じチェックを実行可能にする。

**実装ファイル**: `Makefile`
