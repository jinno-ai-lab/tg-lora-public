## タスク概要

AI Hub feedbackに基づき、advise_training.py CLIのエンドツーエンド統合テストを追加する。合成run_metrics.jsonlファイルを生成し、CLIをサブプロセスで実行し、構造化出力が期待されるadvisory fieldsを含むことを検証する。exit code（0/1/2）の正確性、JSON出力モード、ヘルス判定・アクションリスト・サマリの各フィールドを網羅的にテストする。

## 主要実装項目

### 1. 合成run_metrics.jsonl生成ヘルパー 🔵

**信頼性**: 🔵 *run_metrics.py JSONL形式・TrainingAdvisor入力スキーマより*

各サイクルの損失・勾配ノルム・velocity magnitudeを含む合成JSONLファイルを生成する。収束・発散・停滞の各シナリオに対応する3パターンのヘルパーを提供する。

**実装ファイル**: `tests/test_advise_training_e2e.py`

### 2. E2E正常パステスト 🔵

**信頼性**: 🔵 *test_resume_e2e.pyのCLI subprocess パターンより*

subprocess.run()でadvise_training.pyを実行し、stdoutのJSONを検証。overall_health、actions（優先度ソート済み）、summary、cycle_health、trajectory_summaryの存在と型を確認。

### 3. E2E異常・警告パステスト 🔵

**信頼性**: 🔵 *TASK-0121 exit code仕様（0=正常, 1=ファイルなし/レコードなし, 2=クリティカル状態）より*

存在しないファイルパスでexit code 1、空ファイルでexit code 1、発散データでexit code 2を検証。

## 基本的なテスト要件

### test_advise_training_e2e.py 🔵

**信頼性**: 🔵 *AI Hub feedback + TASK-0121 TrainingAdvisor API仕様より*
