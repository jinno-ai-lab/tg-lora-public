## 実装

### 1. GPU-free dry-run 検証路の defect 修正（fix + test・mutation 証明）

`scripts/run_paper_memory_suite.sh`（`make paper-memory-dry-run` の driver）は `VENV_PYTHON="${VENV_PYTHON:-.venv/bin/python}"` で解決し、dry-run でも per-seed の OmegaConf seed-patch（`${VENV_PYTHON} -c "from omegaconf import OmegaConf..."`）を呼ぶ。fresh clone / CI（`.venv/bin/python` 不在）ではこの行で **exit 127** し、dry-run 自身の validation report に到達する前に crash していた — まさに「GPU 無しで verdict path が assemble するか検証する」tool 自体が壊れていた。

- **fix（`scripts/run_paper_memory_suite.sh`）**: `VENV_PYTHON` が non-executable のとき `command -v python3` へ fallback。real training run は依然 torch/peft/bnb 入り venv を要求するが、dry-run（omegaconf のみ）は `python3` で検証可能。`.venv/bin/python` が存在する環境では byte-identical。
- **test（`tests/test_makefile_targets.py::TestPaperMemoryDryRun`・+1・mutation 証明）**: `VENV_PYTHON` に存在しない path を与え `bash scripts/run_paper_memory_suite.sh` を DRY_RUN で起動し、returncode==0 + `[OK]` + `DRY RUN complete` を表明。**mutation 証明**: fallback guard を `if false` に neutralize すると `assert 127 == 0` で正確に RED・restore で GREEN（test が load-bearing）。
- **検証**: `make paper-memory-dry-run` が exit 0・`DRY RUN complete: all validations passed`。`test_makefile_targets.py` **63 passed**（62 → +1）。
