## 実装詳細

### 1. TrajectoryAnalyzerクラス 🔵

**信頼性**: 🔵 *CycleMonitor・DeltaTrackerの既存パターンに準拠*

`src/tg_lora/trajectory.py`に新規モジュールを作成。学習損失の時系列データをスライディングウィンドウで分析し、収束予測・異常検知・早期停止推奨を提供する。

**実装ファイル**: `src/tg_lora/trajectory.py`

**主要メソッド**:
- `add_point(point: TrajectoryPoint)`: 軌跡点追加
- `compute_loss_trend() -> float`: 直近window件の線形回帰傾き
- `compute_volatility() -> float`: 平均損失変動幅
- `compute_convergence_rate() -> float`: 相対改善率
- `estimate_convergence(target_loss) -> ConvergenceEstimate`: 収束予測
- `detect_anomalies() -> list[str]`: 異常検知
- `early_stop_advice(patience, min_improvement) -> EarlyStopAdvice`: 早期停止推奨
- `full_report(target_loss, patience) -> TrajectoryReport`: 統合レポート
- `from_loss_history(losses, ...)`: ファクトリ（損失リストから）
- `from_dicts(records, ...)`: ファクトリ（辞書レコードから）

### 2. analyze_trajectory.py CLI 🔵

**信頼性**: 🔵 *advise_training.pyのCLIパターンに準拠*

`scripts/analyze_trajectory.py`にCLIツールを作成。run_metrics.json/JSONLまたはカンマ区切り損失リストから軌跡分析レポートを生成。

**実装ファイル**: `scripts/analyze_trajectory.py`

**CLI引数**:
- 位置引数: メトリクスファイルパス
- `--from-losses`: カンマ区切り損失リスト
- `--target-loss`: 収束目標損失値
- `--patience`: 早期停止patience
- `--window`: 分析ウィンドウ幅
- `--output`: 出力ファイルパス
