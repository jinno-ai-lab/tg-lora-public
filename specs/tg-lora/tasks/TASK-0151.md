## タスク概要

spine-anchor ブロックは親文書を markdown link（`[..](../architecture.md)`・ファイル相対）と `parent:` 論理パス（``tg-lora/architecture.md``・`tg-lora/` 名前空間は `specs/` 下に解決）の**2 経路**で再宣言する。両者は同義だが独立に記述されるため、re-point コミントが一方だけ更新すれば silent drift になる。しかも解決規約はコード内に文書化されておらず非自明（`tg-lora/X` → `specs/tg-lora/X`）。これを検査する機構はゼロで、腐敗は手動昇格レビューでのみ発見されていた。

本タスクはこの欠陥クラスを自動 guard で閉じる:

1. **検査 CLI** (`scripts/check_spine_anchors.py`): `specs/`+`docs/` の全 anchor を走査し、(a) begin/end の均衡・link/parent/status フィールド存在、(b) link のファイル相対解決、(c) parent の論理パス解決、(d) **link と parent が同一ファイルに解決されること**（re-point drift の直接検出）を検証。違反があれば stderr に列挙し exit 1。
2. **テスト** (`tests/test_check_spine_anchors.py`): resolver 単体 + clean-tree（0 issue）+ 各 drift モード検出（mismatch/parent_missing/link_missing/no_parent/no_status/no_link/unbalanced）+ CLI 統合（実 repo exit 0・`--root` 外部ツリーで exit 1・`--quiet` 抑制）。drift テストが `render()` の `relative_to` crash（外部 `--root` での真の robustness 欠陥）を露見させ、graceful fallback に修正した。
3. **CI 配線**: `make check-spine` ターゲット + `ci:` パイプラインに挿入。「手動レビュー依存」→「CI 自動 gate」。
