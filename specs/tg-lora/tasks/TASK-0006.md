## タスク概要

学習中のメトリクスを構造化して出力する機能を実装する。サイクルごとの損失・reduction_rate・acceptance_rate・ハイパーパラメータ推移を CSV/JSON で記録し、学習終了時にサマリーレポートを生成する。

## 完了条件

- [ ] サイクルごとのメトリクスが CSV ファイルに追記される
- [ ] 全サイクル完了後に JSON サマリーが生成される
- [ ] メトリクス内容: cycle, train_loss, valid_loss, reduction_rate, acceptance_rate, K, N, alpha, beta, lr, accepted
- [ ] ログ出力が `tg-lora` ロガーで構造化される
- [ ] 学習終了時にテキスト形式のサマリーが標準出力される

## 主要実装項目

### MetricsRecorder クラス 🔵

**信頼性**: 🔵 *CycleState.summary() のデータ構造に基づく*

各サイクルの終わりに以下を記録:

- cycle 番号
- train_loss, valid_loss
- reduction_rate, acceptance_rate
- 現在の K, N, alpha, beta, lr
- accepted (bool)
- active_layer_strategy
- velocity_magnitude

**実装ファイル**: `tg_lora/metrics.py`

### CSV 出力 🔵

**信頼性**: 🔵 *Python csv モジュールの標準パターン*

`metrics_{timestamp}.csv` にヘッダー付きで 1 行ずつ追記。学習再開時は既存ファイルに追記。

**実装ファイル**: `tg_lora/metrics.py`

### JSON サマリー 🔵

**信頼性**: 🔵 *CycleState.summary() の構造に基づく*

学習終了時に `summary_{timestamp}.json` を生成。最終的な全メトリクスの集計値を含む。

**実装ファイル**: `tg_lora/metrics.py`
