## 背景（torch/checkpoint/ledger は atomic、JSON deposit だけ bare `open("w")` だった）

repo の atomic-write 規律は厳格:
- `src/utils/atomic_save.py::_atomic_torch_save`（temp + `os.replace`・唯一の `torch.save` publish point・2つの AST guard で pin）
- `src/utils/checkpoint.py`（atomic dir rename）・`src/utils/checkpoint_integrity.py`（load 側 torn 検出）
- ledger（`append_arm_to_ledger` = append-only JSONL・arm 単位で crash-safe）

だが **citable §4 verdict deposit JSON**（多時間・多 arm run の最重要出力・全 arm 完了後 1 回だけ書かれる）と **run-log witness**、そして io-leaf の `save_json`/`save_jsonl`（eval 予測・prefix cache summary・data records に使用）は **bare `open(path, "w")`（truncate-then-write）** だった。よって書き込み中の kill（OOM-kill・SIGINT・host の worktree recycling）や **harvest-during-write**（`cp …/freeze_validloss_ci_9b_full_heterogeneous.json tests/fixtures/` が書き込み中に走る）が **torn deposit** を残す: parse 失敗（= run 無駄）か、最悪、parse できるが silent-truncation された verdict（= §7 honesty 違反・誤番号 commit）。これは着地前の run-time OOM（TASK-0162）とは別の、**着地時** integrity bug。`atomic-torch-save-axis-complete` は `torch.save` のみを扱い JSON write は未カバーだった。

## 実装

`src/utils/io.py` に `_atomic_write_bytes(path, data)` + `_atomic_write_text(path, data, *, encoding="utf-8")` を追加（`_atomic_torch_save` と同形: same-dir PID-suffixed temp → write → `os.replace` → `except BaseException` で orphan temp cleanup + re-raise。`BaseException` なので SIGINT/SystemExit も cleanup）。`save_json`/`save_jsonl` を `_atomic_write_bytes` 経由に（byte-identical・root 化）。

`scripts/run_freeze_validloss_ci_9b.py`: `_write_run_log` と新設 `_write_deposit(path, payload)` helper を `_atomic_write_text` 経由に。`main()` の deposit write（旧 bare `open("w")`）を `_write_deposit` 経由に。
