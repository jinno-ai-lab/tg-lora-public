## タスク概要

`compare_freeze_levels()`（GOAL §5 / Phase 3 の Level-1 vs Level-2 定量比較）は、ヘッドライン量 `additional_realized_reduction`（Level 2 の suffix 切断が Level 1 基盤の上で追加実現する後方削減）を proxy 算術からの**点推定**として報告していた。§6.3 の `ConfidenceBand` / `ReductionSample` / `calibrate_reduction_band` / `per_cycle_realized_reductions` / `from_runs` 機構は存在し、パイプライン（`analyze_dynfreeze_experiment.py`）は単一 level の per-cycle spread に band を較正していたが、**Phase 3 の A/B ヘッドラインを再生産 N 件でまたぐ測定分散を受け取る着地点**が存在せず、実 A/B run の観測を反映する経路がなかった。AI Hub steering（bullet 2）も同じ指摘: 「thin (N=2) bracket を十分な再生産数で厚くせよ」。

本タスクは、その bracket を**監査可能な証拠経路**として実装する。`ReproductionRecord` が N 件の実測ヘッドライン削減（観測値・`source`）を運び、`calibrate_reproduction_bracket()` が既存の `ReductionSample` / `calibrate_reduction_band` に委譲して §6.3 `ConfidenceBand` を較正し、`LevelComparison.reproduction_bracket` に載せ、`format_level_comparison()` が監査行として出力する。既定（record なし）は `None` で**出力は byte-identical**、薄い証拠（`MIN_SAMPLE_FOR_CONFIDENCE_BAND` 未満）は `THIN_EVIDENCE` + 件数を明示する。これは §7 判定を反転させない**不確実性報告**であり（per-level band と同じ honesty 役割をヘッドラインについて担う）、gate の厳格化ではない — steering が「既に厳格な gate をこれ以上硬化するな」と明示した点にも合致する。実測値の供給は GPU 依存の別作業であり、本タスクはそれを受け取る pluggable な着地点のみを提供する。

## 主要実装項目

### 証拠レイヤ（ReproductionRecord / calibrate_reproduction_bracket）🔵

**信頼性**: 🔵 *§6.3 design「per-cycle / per-run 観測を ReductionSample に蓄積して band を較正」+ ReproductionRecord の §6.3 honesty 哲学 + 既存 calibrate_reduction_band への委譲より*

- `ReproductionRecord` は `Level1RealizationRecord`（§6.2 ceiling）と対をなす §6.3 着地点: N 件の実測ヘッドライン削減と `source`（例 `9b_ab_4runs`）を運び、bracket の根拠を監査可能にする。ceiling が *realizability* を、bracket が *measurement spread* を境界づける（直交）。
- `calibrate_reproduction_bracket` は `calibrate_reduction_band` に丸ごと委譲し、empirical_envelope / normal の両方と z・method の検証を再実装せず再利用する。None のときだけ `None` を返す分岐が、既定の byte-identical 契約を担う。

### 比較への伝播（reproduction_bracket / reproduction_record）🔵

**信頼性**: 🔵 *compare_freeze_levels / LevelComparison / format_level_comparison の既存契約（TASK-0132 の level1_ceiling / level1_record 伝播と同形）より*

- 実験レイヤ（`compare_freeze_levels`）は record オブジェクト `reproduction_record` を受け取り1度解決する。算術レイヤは触れない（bracket は §7 判定に影響しない）。
- `LevelComparison.reproduction_bracket`（既定 `None`）が「点推定か、N 件で較正された bracket か」を監査に残す。`format_level_comparison` は `None` のとき行を出さず（byte-identical）、薄い証拠のとき `THIN_EVIDENCE` + 件数を行に出す。
