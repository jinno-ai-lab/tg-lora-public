## タスク概要

GPU環境（RTX3060 12GB）でQwen3.5-9Bモデルの4bit QLoRA読み込みを検証し、学習データの準備状況を確認する。学習スモークテストの前提条件を整える。

## 依存タスク

- **前提タスク**: [TASK-0020](TASK-0020.md) (docs/llm-wikiクリーンアップ推奨)

## 完了条件

- [x] `torch.cuda.is_available()` がTrueを返す
- [x] `nvidia-smi` で12GB以上のVRAMが確認できる
- [x] Qwen3.5-9Bの4bit量子化モデル読み込みが成功する
- [x] LoRA adapter適用後のパラメータ数が想定通り
- [x] data/train.jsonl, data/valid_quick.jsonl, data/valid_full.jsonlが存在する
- [x] モデル+LoRAのVRAM使用量が10GB以下に収まる

## 主要実装項目

### 1. GPU環境確認 🔵

**信頼性**: 🔵 *AGENTS.md RTX3060 12GB仕様より*

CUDA利用可能性、VRAM容量、driver/bitsandbytes互換性を確認。

### 2. モデル読み込み検証 🔵

**信頼性**: 🔵 *load_model.py既存実装・9b_tg_lora.yaml設定より*

`load_base_model(cfg)` → `apply_lora(model, cfg)` の実行。4bit NF4量子化 + LoRA r=16 でVRAM使用量を確認。

**実装ファイル**: `src/model/load_model.py`

### 3. データ準備確認 🔵

**信頼性**: 🔵 *download_data.py・prepare_data.py既存実装より*

`make download-data && make prepare-data` で学習データが準備済みであることを確認。ファイル存在・レコード数の検証。
