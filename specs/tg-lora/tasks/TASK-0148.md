## タスク概要

TASK-0146/0147 は top-level で `from src.* import` する7 script が bare CLI として起動できず、対応 CLI/E2E test を import 段で死なせていた欠陥を、repo-root を `sys.path` に載せる1行 idiom で閉じた。しかしその1行欠陥は **まだ閉じていなかった**: top-level で `from src.* import`（または sibling `from scripts.compare_runs import`）する別の8 script が同一の「bare 起動で `ModuleNotFoundError` 即死」状態にあり、`test_cli_help_smoke.py` の `--help` smoke case を不活性化していた（test 上は FAIL で見えていたが、本来 CI 通過を主張すべき発射性が静かに壊れていた）。

これは TASK-0140〜0147 が一貫して排除してきた「コードは直ったように見えるが実は不活性」という静かな状態の**発生源の bootstrap-fixable な残り全部**である。特に `diagnose.py` / `recover.py` は checkpoint 復旧・診断の運用入口、`export_paper_results.py` は論文成果物出力入口、`benchmark_prefix_cache.py` は性能測定入口であり、これらが `--help` すら到達しない状態だった。

本タスクは TASK-0146/0147 と同一の1行 idiom を8 script に機械適用し（`import sys` / `from pathlib import Path` が未 imported の script には併せて追加）、発射性を自己完結的に回復する。新規依存・新規 knob なし。うち7 script は bare `--help` が発射して smoke が緑化する。残1 script（`benchmark_optimizer_lifecycle.py`）は bootstrap が path 欠陥を閉じたことがエラー進行（`src.model` → `peft`）で確認できたが、transitive 依存 `peft` が本 worktree の pyenv env に未導入のため `--help` 到達は env 側の依存導入待ちとなり、本タスクはこれを明示的に scope 外として正直に記録する（bootstrap 自体は正しく必要な修正として残す）。

## scope 外（bootstrap 単独では解決しない残り・正直な取り扱い）

`test_cli_help_smoke.py` は本タスク後も以下4 case で FAIL が残る。いずれも **bootstrap とも1行コード欠陥とも無関係**であり、本タスクで触れない（依存導入・private pipeline 創設は環境/別タスクの作業）:

- `benchmark_optimizer_lifecycle.py` → bootstrap 適用済み（path 欠陥は閉鎖）。しかし `src.model.load_model` が `from peft import ...` し、`peft`（pyproject 依存 L13）が本 worktree の pyenv env に未導入のため `ModuleNotFoundError: No module named 'peft'`。**bootstrap 修正は正しい（peft 導入後は bare 発射する）ため残置**。`peft` 導入は環境側の作業。
- `download_data.py` → `ModuleNotFoundError: No module named 'datasets'`。`datasets` library が本 env に未導入。
- `prepare_data.py` → `ModuleNotFoundError: No module named 'src.data'`。
- `precompute_prefix_cache_parallel.py` → `ModuleNotFoundError: No module named 'src.data'`。
- 後者2件は private `src.data` pipeline が public mirror に欠損していることが原因（[[public-mirror-preexisting-test-failures]] 既知制約）。bootstrap を載せても次行で同一 import 死となるため、`src.data` 創設または当該 case の smoke skip 化が必要であり、本タスクの1行 idiom の scope を超える。
