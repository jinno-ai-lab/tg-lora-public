## 背景（feedback の指示と、本 mirror で取り得る SINGLE highest-leverage 実証）

AI-Hub feedback は「9B run を stall している standing excuse は消えた。次 iter は GPU driver を `configs/9b_baseline_suffix_only_last25.yaml` に bind し real 9B target-scale A/B verdict を出せ。scaled-smoke や gate polish にもう 1 iter 使うな」と指示した（前 iter VALUABLE 評価）。TASK-0154 はこの verdict を done / blocked-deferred / blocked-research の 3 leg に形式化し、blocked-deferred leg の根拠を「9B QLoRA は 12GB で seq≥512 OOM（実証済）」とした。

**本 iter の問題意識**: TASK-0154 の OOM 根拠は **standard config（`9b_baseline`・`trainable_lora_scope: all`）でのみ** 計測された。feedback が bind を指示した **suffix-only config（`last_25_percent`・gradient checkpointing on）** は LoRA 勾配/optimizer state を ~4x 削減し、かつ gradient checkpointing を有効化する — すなわち **target scale で一度も計測されていない異なる memory 特性** を持つ。「suffix-only も OOM する」という standing excuse は**未計測の推測**であって、誠実性（§7）上 form する前に計測で置換しなければならない。

完全 §4 verdict（seq≥1024 + valid_full・loss 値）は `src/data/build_seed_dataset.py`（private・本 mirror に不在）を要するため、**loss 軸**は本 mirror では取得不能（TASK-0154 REQ-004）。しかし **memory 軸**（その verdict が物理的に生産可能か）は data pipeline から独立であり、trainer の public model-loading path（`load_base_model` → `apply_lora` → `configure_trainable_lora_scope`）を再利用し data-loading のみを synthetic batch で bypass すれば、real `Qwen/Qwen3.5-9B` 上で **data-independent に** 計測できる。これが本 iter の SINGLE highest-leverage 実証である。

## 実装

### 1. 経験的 memory-frontier probe（`scripts/probe_9b_memory_frontier.py`）

data-independent に「本 config の memory stack が 12GB で seq_len≥1024 を fit させるか」を問う単一の probe。trainer の**正確な public model-loading path** を再利用し、private `src.data` の data-loading のみを bypass（fully-supervised synthetic batch・`labels==input_ids`・CE loss は HF-internal `model(labels=...).loss` = 文書化された OOM driver である `logits.float()` upcast を含む）。すなわち real trainer が 1 step で hit するのと同一の per-step peak を計測する。

- **pure/orchestrable core**: `sweep_seq_lens`（OOM を `PeakMemoryOOM` 例外で catch し `fit=False` として記録・larger seq も skip せず記録）・`max_fit_seq_len`・`full_section4_producible`（threshold=1024・未計測は `None` = "unmeasured" であって "failed" ではない）。
- **GPU step-runner**: `make_gpu_step_runner` は forward+backward+AdamW の per-step peak を reset 直後に計測（model + optimizer state resident 済みの high-water mark）。
- **`FULL_SECTION4_SEQ_LEN = 1024`**（GOAL §4 / TASK-0154 REQ-003）。

### 2. 単体 test（`tests/test_probe_9b_memory_frontier.py`・34 passed・mutation 証明）

GPU 不要。`sweep_seq_lens` の OOM-catch が load-bearing であることを fake step-runner で証明、`full_section4_producible` の unmeasured=`None` vs failed=`False` の区別、`make_gpu_step_runner` が OOM を `PeakMemoryOOM` に lift することを直接検証、`FrontierResult.to_dict` の JSON 往復、等。

### 3. real 9B GPU 計測 + replay faithfulness test

- **real 計測**（`make probe-9b-memory`・upstream venv `/home/jinno/tg-lora/.venv`）: 両 arm で frontier 取得 → `tests/fixtures/probe_9b_memory_frontier.json` に deposit。
- **replay test**（`TestFixtureReplay`・+5）: fixture の生 measurements から `max_fit_seq_len`/`full_section4_producible` を pure 関数で再導出し、**GPU 無しで** 記録 verdict との faithfulness を pin。**mutation 証明**: fixture の seq1024 `fit` を反転させると `assert max_fit_seq_len == 1024` が正確に RED（test が load-bearing）。

### 4. Makefile target（`make probe-9b-memory`）

`PROBE_CONFIG`/`PROBE_SEQ_LENS`/`PROBE_BATCH_SIZE`/`PROBE_OUTPUT` 変数（Makefile global の `CONFIG`/shell env `SEQS` との衝突を回避）で `configs/9b_baseline_suffix_only_last25.yaml` に bind。

## 計測結果（controlled A/B・12GB RTX 3060・`Qwen/Qwen3.5-9B` 4-bit nf4・bfloat16 compute・double-quant・gradient checkpointing）

| arm | config | scope | trainable params | seq1024 | max fit seq |
|---|---|---|---|---|---|
| **suffix_only_last25** | `9b_baseline_suffix_only_last25.yaml` | `last_25_percent` | 10,819,584 | **FIT**（11226 MiB / 11910 MiB・684 MiB headroom） | **1024** |
| standard_all | `9b_baseline.yaml` | `all` | 43,278,336 | OOM | 768 |

**controlled contrast**: model + GPU + 4-bit quant + gradient checkpointing を固定し、唯一 `trainable_lora_scope`（`last_25_percent` vs `all`）のみを変える。standard は seq1024 で OOM・suffix-only は seq1024 を **684 MiB の headroom を残して fit**。trainable params は 43,278,336 → 10,819,584（4.0x 削減）。seq≥1280 は suffix-only でも OOM（完全 2048 context は >12GB を要す）。
