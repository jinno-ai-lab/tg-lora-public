## 背景（`float()` が NaN/inf を黙って受理し verdict を汚染する）

`scripts/form_freeze_validloss_deposit.py::extract_best_valid_loss` は upstream `run_metrics.jsonl` の各 line から `best_valid_loss`（footer）と `loss_valid` / `loss_valid_full`（step）を読み、`float()` で数値化して deposit の `candidate_losses` / `surrogate_losses` 配列に格納する。これらの配列は :func:`surrogate_valid_loss_ci` の bootstrap CI に直接流入し §4 verdict（SURPASSES/TIES）を決定する。

`float()` は `"nan"` / `"inf"` / `"-inf"` を**黙って受理**する（例外を投げない）。QLoRA 学習の gradient explosion は正に `loss_valid` / `best_valid_loss` を NaN/inf にする定型 failure mode である。ゆえに diverged run の artifact を deposit 形成に流すと:

- **step 経路**: 最初の有限値より前に NaN step が現れると `min_step_loss = float("nan")` で seed され、以後の比較 `candidate < nan` は常に `False` となり min が NaN に固定 → NaN が返る。
- **footer 経路**: `best_valid_loss: NaN` は `float(NaN) != None` で footer 値として採用 → NaN が返る（position 非依存・1 line で発火）。

返った NaN は配列に入り `numpy.mean([1.5, nan]) == nan` → NaN CI → verdict 比較（`nan < 0` は `False`）が garbage / 誤分類 → **corrupt-but-green §4 verdict**（GOAL §7）。これは `7c4aebf` が閉じた「非-Dolly schema が空文字 `.get` → 不正 float → corrupt verdict」と**同一 class** の silent-corruption である（入力 dataset 側 vs 出力 metrics 側の違いのみ）。`isfinite` / `isnan` / `math.is` による防衛は本 verdict 経路のいずれの file にも存在しない（grep 0 = 本 iter で初閉鎖）。

## 実装（`scripts/form_freeze_validloss_deposit.py`）

非有限値を既存の `None`「信号無し」経路と同一に扱う helper を追加し、step 読み取りと footer 読み取りの両方の `float()` 適用点に適用:

```python
def _finite_loss_or_none(value: Any) -> float | None:
    f = float(value)
    return f if math.isfinite(f) else None
```

- **step 経路**: `loss_valid` / `loss_valid_full` の各 record で `_finite_loss_or_none(raw)` を通し、`None`（非有限）なら既存の missing-field と同じく skip。偶発的な glitch step を持つ run は honest な有限 minimum を保ち、NaN で min が固定されることはない。
- **footer 経路**: `best_valid_loss` も `_finite_loss_or_none` を通し、非有限なら「usable footer 値無し」として step-min fallback に fall through（`null` footer と同扱い）。run 全体が非有限（全 step + footer が NaN/inf）なら既存の loud `ValueError("no best_valid_loss ...")` が発火し、broken run が表面化する。

保証: **非有限 float は deposit の losses 配列に絶対に入らない**。これは verdict logic や citation gate に手を入れず、形成 chokepoint（`float()` 適点）のみで閉じる最小修正。`replay_freeze_valid_loss_ci` 側への第二防衛は意図的に置かない（同一軸への N+1 guard になるため・PURPOSE.md の "honesty-guard N+1 を足し続ける" 失敗モード回避）。

## test（`tests/test_form_freeze_validloss_deposit.py`・mutation 証明）

非有限閉鎖 block として 4 test を追加（`_write_run_metrics` helper が `json.dumps(float("nan"))` → `NaN` → `json.loads` → `nan` の round-trip を利用）:

1. `test_extract_skips_non_finite_footer_uses_step_min` — footer `best_valid_loss=NaN` + 有限 step `[1.30, 1.06, 1.18]` → 有限 step min `1.06` を返し `math.isfinite(value)` が真・source は `min_loss_valid_step`（NaN footer が採用されない）。**primary mutation proof・position 非依存**。
2. `test_extract_skips_non_finite_step_losses` — step `[nan, 1.50, inf, 1.10]`・footer 無し → `1.10`。NaN が**先頭**にあるので guard 無しでは `min_step_loss` が NaN に固定される（以後の `x < nan == False` で更新されない）。
3. `test_extract_raises_when_entire_run_non_finite` — footer `NaN` + 全 step `[nan, inf, -inf]` → `ValueError("no best_valid_loss")`（loud fail・空 file と同契約）。
4. `test_form_deposit_never_emits_non_finite_losses` — form_deposit E2E・1 candidate が diverged（NaN footer + 有限 step）でも candidate_losses/surrogate_losses が全て有限で、diverged arm は有限 step min `1.20` を寄与（P0 deposit-integrity property の end-to-end pin）。

**mutation 証明**: `_finite_loss_or_none` の `return f if math.isfinite(f) else None` を `return f`（finite-check 無力化 = pre-fix の bare `float()` と等価）にすると、上記 4 test 全てが RED（既存 23 test は deselected で mask 無し）・復元で GREEN。各 test が load-bearing であることを確認。
