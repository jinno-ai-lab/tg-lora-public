## 背景（deposit が stdout と同じ payload を書くと `--output` 単独で text deposit を生む）

`main()` は deposit と stdout を**同一 payload** から書いていた:

```python
payload = json.dumps(...) if args.json else format_report_9b(result)
print(payload, flush=True)
if args.output:
    _write_deposit(args.output, payload)
```

`--json` が stdout のみを制御する意図（[TASK-0162](TASK-0162.md) の exit-code contract docstring も `--json` = "emit JSON evidence to stdout"）にもかかわらず、`--output` はこの payload をそのまま書くため、**`--output` を `--json` なしで渡すと citable verdict deposit に人間可読 text report が書かれる**。これは:

1. **§7 honesty 違反**: deposit は harvest されて `tests/fixtures/` に置かれ JSON として re-load（`TestDepositGateSelfConsistency` / pin test 群が `json.loads`）される。text report は `json.loads` で parse 失敗 = run が無駄になるか、最悪 parse できるが verdict を欠いた corrupt deposit。
2. **silent**: 既存 caller（Makefile `--json --output` 常に pairing・launcher も同様）が text deposit を踏まないため test が green のまま潜在化。TASK-0163 の atomic write は torn-write を防ぐが**内容が text である**ことは防がない。`_write_deposit` は TASK-0163 で atomic に なったが、**渡された `payload` が JSON である保証は呼び出し側の convention のみ**（= guard なし・drift すれば silent）。

## 実装

`scripts/run_freeze_validloss_ci_9b.py::main()`: deposit と stdout を別 payload に decouple。

```python
# deposit は常に canonical JSON (result_to_json)・`--json` は stdout のみを制御。
# `--output` (deposit) は JSON 無条件 — harvest 後 JSON として re-load されるため
# text report は silently-corrupt verdict になる (same §7 class as 54a4cd8/fingerprint)。
deposit_payload = json.dumps(result_to_json(result), indent=2)
stdout_payload = deposit_payload if args.json else format_report_9b(result)
print(stdout_payload, flush=True)
if args.output:
    _write_deposit(args.output, deposit_payload)
    logger.info("Wrote %s", args.output)
return 0
```

`--output` の argparse help も「常に canonical JSON shape・`--json` から独立（`--json` は stdout の JSON vs text のみ選択）」に整合。
