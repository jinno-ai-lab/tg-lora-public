## 目標

hypothesisを使用してRandomWalkController.accept()のプロパティベーステストを追加する。相対許容誤差の大きさ依存性を広範囲に検証する。

## 背景

コミット(02582db)でaccept()が絶対許容誤差から相対許容誤差に変更された。これによりloss_pilotの大きさに依存する挙動が導入された。AI_HUB_MAKE_RUN_FEEDBACKの指摘「Consider extending the relative-tolerance accept() change with a property-based test (e.g., hypothesis) that verifies idempotence across loss-value magnitudes」に対応する。

## 要件

- REQ-071: accept()プロパティベーステスト
- EDGE-136: accept()相対許容誤差大きさ依存性

## 前提条件

- hypothesisの依存追加（pyproject.toml）が必要
  - `[project.optional-dependencies]` の `dev` に `"hypothesis>=6.0"` を追加

## テストケース

### プロパティベース

1. **TC-071-P19-01**: loss_after <= loss_pilot は大きさに関わらず常にTrue
   - hypothesis: st.floats(1e-6, 1e6), 改善率 st.floats(0, 1.0)

2. **TC-071-P19-02**: NaN/Inf入力は常にFalse
   - hypothesis: st.one_of(st.just(float("nan")), st.just(float("inf")), ...)

3. **TC-071-P19-03**: 相対許容誤差の大きさ一貫性
   - hypothesis: 同じ改善率で異なる大きさのlossペア

4. **TC-071-P19-04**: 1e-8 floor近傍での挙動確認
   - hypothesis: loss_pilot ~ 1e-8の近傍値
