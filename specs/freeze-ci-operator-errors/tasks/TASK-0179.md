## 背景（現状の entrypoint は 4 種類の operator error を distinct に表現しない）

`scripts/replay_freeze_validloss_ci.py` / `scripts/run_freeze_validloss_ci_9b.py` / `scripts/launch_freeze_ci_9b_full.py` は operator が投入する **4 種類の distinct な error**（`--config X` で `X` 欠落 / 既存 `X` の YAML 壊れ / YAML は parse 可だが Pydantic schema 違反 / `replay` script の `<samples_file>` schema 違反）を、現コードでは:

- `FileNotFoundError` が raw で `python -m` の traceback 末尾を汚す
- `yaml.YAMLError` が stderr に bare で流れる（行/列情報は保持されるが class 識別不能）
- `pydantic.ValidationError` が複数 field 違反を `errors()` 配列で dump するだけで「最初の 1 件」が operator に届かない
- `json.JSONDecodeError` / 必須 key 欠落 / 型不一致 が混在して "eval results 形式が想定外" としか分からない

これにより **CI log の grep・operator による 1 回目修正・launcher の exit code 分類** のすべてが成立しない（CI log を見ないと「設定 typo か・YAML 文法か・schema か・eval data か」を区別できない）。AI_HUB_MAKE_RUN_FEEDBACK は明示:

> Use the newly established witnesses as the boundary for an operator-facing follow-up: implement distinct handling for missing config, malformed YAML, AppConfig validation failures, and malformed eval results with specified messages and exit statuses, then retain these tests as regressions.

本 TASK はその **leaf 側 prereq**（chokepoint の最小核）を切り出す。3 entrypoint の wrapping（TASK-0180..0182）は本 TASK の `raise_*` / `emit_operator_error` / `EXIT_OPERATOR_ERROR` を使って薄い outer try/except を張るだけになる。

## 設計（leaf module 集約 + zero-dep + 4 wrapper raise-only）

`interfaces.py` の signature を逐語で実装する。重要 point:

- **zero-dep leaf**: `yaml` / `pydantic` / `omegaconf` / `torch` を import しない（`interfaces.py` の `TYPE_CHECKING` と同様）。`Path` だけは local import（`is_dir()` 検出のみ）。`import json` / `import sys` は `emit_operator_error` 内で local import。
- **4 subtype は `OperatorError` の subclass のみ**。`yaml.YAMLError` / `pydantic.ValidationError` は **wrap のみ**（subclass 化しない = leaf が heavy 依存を drag しない）。
- **4 wrapper は `NoReturn` 型**。`raise` だけして返らない（caller の flow を中断）。
- **emitter は `json_mode` flag** で stderr (human) / stdout 1 行 JSON (--json) を切替。`stdout` / `stderr` は Protocol 型 `_SupportsWriteText` で受け、production では `sys.stdout` / `sys.stderr`、test では `io.StringIO` 注入。
- **`EXIT_OPERATOR_ERROR = 78`**: `sysexits.h` `EX_CONFIG` 由来。`argparse.error` exit 2 / `ad8c84a` の 4 worker exit code（0/1/2/3/75）と **衝突しない独立値**。
- **message format pin**: `to_dict()` は `{"error": type(self).__name__, "detail": self.detail, "exit_status": self.exit_status}` 固定（EDGE-102 で JSON が 1 行・末尾改行 1 個のみ）。`__str__()` は `f"{type(self).__name__}: {self.detail}"` 固定（NFR-202 で class 名 prefix・grep 抽出可）。

## feat

`src/utils/cli_errors.py`（NEW・interfaces.py 逐語実装）:

1. `EXIT_OPERATOR_ERROR: int = 78`（module-level 定数）
2. `class OperatorError(Exception)`:
   - `default_exit_status: int = EXIT_OPERATOR_ERROR`
   - `__init__(self, detail: str, *, exit_status: int | None = None)` — `super().__init__(detail)` + `self.detail` + `self.exit_status`（REQ-301a の per-instance override hook・本 TASK では default 78 のみ使用）
   - `__str__(self) -> str` — `f"{type(self).__name__}: {self.detail}"`（NFR-202 prefix 一致）
   - `to_dict(self) -> dict[str, Any]` — `{"error": type(self).__name__, "detail": self.detail, "exit_status": self.exit_status}`（EDGE-102 1 行 JSON 契約）
3. `class MissingConfigError(OperatorError)` / `class MalformedYAMLError(OperatorError)` / `class AppConfigValidationError(OperatorError)` / `class MalformedEvalResultsError(OperatorError)`（4 subtype・空 body）
4. `def raise_missing_config(path, *, kind: str = "config") -> NoReturn`:
   - `p = str(path)`
   - `try: from pathlib import Path; if Path(p).is_dir(): suffix = " (is a directory)"`
   - `except OSError: pass`（broken symlink / permission error → missing 扱い）
   - `raise MissingConfigError(f"{kind} not found: {p}{suffix}")`（REQ-101 / REQ-102 / EDGE-001）
5. `def raise_malformed_yaml(path, exc: "yaml.YAMLError") -> NoReturn`:
   - `raise MalformedYAMLError(f"yaml parse error in {path}: {exc}")`（REQ-201 / REQ-202 — PyYAML `__str__` の行/列情報保持）
6. `def raise_app_config_validation(config_class: str, exc: "pydantic.ValidationError") -> NoReturn`:
   - `errors = exc.errors()`; `n = len(errors)`; `first = errors[0] if errors else {"loc": (), "msg": "unknown", "type": "unknown"}`
   - `loc = ".".join(str(x) for x in first.get("loc", ())) or "<root>"`
   - `raise AppConfigValidationError(f"schema validation failed for {config_class}: {n} errors; first: {loc} {msg} ({typ})")`（REQ-301..303 — `error_count` + first error triple）
7. `def raise_malformed_eval_results(reason: str, detail: str = "") -> NoReturn`:
   - `if detail: raise MalformedEvalResultsError(f"{reason}: {detail}")`
   - `else: raise MalformedEvalResultsError(reason)`（REQ-401..402 / EDGE-003..004 — "missing key: candidate_total" / "invalid type for samples: expected list, got str"）
8. `def emit_operator_error(exc: OperatorError, *, json_mode: bool = False, stdout=None, stderr=None) -> None`:
   - `out = stdout if stdout is not None else sys.stdout`; `err = stderr if stderr is not None else sys.stderr`
   - `if json_mode: out.write(json.dumps(exc.to_dict()) + "\n"); out.flush()`
   - `else: err.write(str(exc) + "\n"); err.flush()`（REQ-501 / EDGE-102 1 行 / EDGE-103 ANSI なし）

保証:
- 4 subtype import 可・`isinstance(e, OperatorError)` True・`to_dict()` 固定 schema（TC-001-01..03 合格）
- 4 wrapper は `NoReturn`・`raise` のみ・caller flow 中断（TC-NFR-101-01 mutation 証明）
- emitter は `json_mode` で stdout 1 行 JSON / stderr 空、それ以外で stderr class prefix・stdout 無干渉（TC-NFR-201/202 合格）
- leaf import 副作用なし（torch / pydantic / omegaconf / yaml を import しない）— `python -c "import src.utils.cli_errors"` が import error なしで完了
